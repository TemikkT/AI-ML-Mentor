# PyTorch — базовое руководство

## Что такое PyTorch?

PyTorch — это библиотека (фреймворк) для глубокого обучения, которую используют исследователи и инженеры по всему миру. Её главная фишка — она работает почти как NumPy, но при этом умеет автоматически считать градиенты (это нужно для обучения нейронок) и легко переключается на видеокарту (GPU), что даёт ускорение в десятки раз.

В центре всего находятся **тензоры** — это те же самые матрицы и многомерные массивы, как в NumPy, но с дополнительными возможностями: они могут лежать на GPU и автоматически запоминать, какие операции с ними делали, чтобы потом посчитать производные.

---

## 1. Тензоры — основа всего

Прежде чем мы начнём делать нейронные сети, нужно понять, с чем мы вообще работаем. В PyTorch все данные — это тензоры. Что такое тензор простыми словами? Это многомерный массив чисел. Если у вас одна колонка чисел — это 1D тензор (вектор). Таблица с row и col — 2D тензор (матрица). Пачка картинок — 4D тензор (batch × высота × ширина × каналы).

Давайте разберёмся, какие типы тензоров бывают и зачем:

- **FloatTensor (torch.float32)** — основной тип для нейронок. Веса модели, входные данные — всё хранится в float32. Почему не float64? Потому что веса не нужно считать с точностью до 15 знаков, а памяти float32 занимает в 2 раза меньше.
- **LongTensor (torch.int64)** — для меток классов (0, 1, 2...). Функции потерь вроде CrossEntropyLoss требуют именно int64 на вход.
- **BoolTensor (torch.bool)** — для масок. Например, `x[x > 0]` возвращает маску true/false.

### Создание тензоров

Самый простой способ — из списка:

```python
import torch

# Из списка Python
x = torch.tensor([[1, 2], [3, 4]], dtype=torch.float32)
print(x)
# tensor([[1., 2.],
#         [3., 4.]])
```

Но на практике чаще используют функции, которые сразу создают тензоры нужной формы и типа:

```python
# Заполненные константами — удобно для тестов
ones = torch.ones(2, 3)       # матрица 2×3 из единиц
zeros = torch.zeros(2, 3)     # матрица 2×3 из нулей
eye = torch.eye(3)            # единичная матрица 3×3

# Случайные числа — для инициализации весов
rand = torch.rand(3, 3)       # равномерное распределение [0, 1)
randn = torch.randn(3, 3)     # нормальное распределение (среднее=0, дисперсия=1)

# Числовой диапазон — как range(), только тензор
arange = torch.arange(0, 10, 2)     # [0, 2, 4, 6, 8]
linspace = torch.linspace(0, 1, 5)  # [0.0, 0.25, 0.5, 0.75, 1.0]

# Из NumPy массива — если данные уже в numpy
import numpy as np
np_arr = np.array([[1, 2], [3, 4]])
tensor_from_np = torch.from_numpy(np_arr)  # разделяют память с numpy!
```

**Важный момент про torch.tensor() vs torch.Tensor():**

- `torch.tensor([1, 2])` — создаёт тензор, копируя данные. Он сам подбирает тип или использует ваш dtype.
- `torch.Tensor([1, 2])` — это псевдоним для `torch.FloatTensor`. Всегда создаёт float32.
- Лучше всегда использовать `torch.tensor()` с явным `dtype` — меньше сюрпризов.

### Размерность: .shape, .size(), .ndim, .numel()

Размер тензора — это, пожалуй, самая важная характеристика. Ошибки с размерностью — самая частая проблема новичков.

```python
x = torch.randn(2, 3, 4)  # 3D тензор: 2 "слоя", 3 строки, 4 столбца

print(x.shape)      # torch.Size([2, 3, 4]) — полный размер
print(x.size())     # то же самое
print(x.ndim)       # 3 — количество измерений
print(x.numel())    # 24 — общее количество элементов (2*3*4)
```

Как читать shape слева направо: первое измерение — батч (сколько независимых примеров), последнее — признаки (что мы знаем об одном примере). То, что посередине — пространственная структура (пиксели, токены и т.д.).

### Изменение формы: .reshape(), .view(), .transpose()

В нейронках постоянно нужно менять форму тензоров. Например, свёртка выдала 4D [batch, channels, h, w], а полносвязный слой ждёт 2D [batch, features].

```python
x = torch.randn(4, 3, 32, 32)  # батч из 4 картинок RGB 32×32
print(x.shape)  # [4, 3, 32, 32]

# reshape — меняет форму, не меняя данные
x_flat = x.reshape(4, -1)  # -1 значит "подбери размер автоматически"
print(x_flat.shape)  # [4, 3072]  (3*32*32 = 3072)

# view — как reshape, но быстрее (но не всегда работает)
x_flat2 = x.view(4, -1)  # работает, только если данные в памяти лежат подряд
print(x_flat2.shape)  # [4, 3072]

# transpose — перестановка измерений
x_t = x.transpose(1, 2)  # меняем местами измерение 1 и 2
print(x_t.shape)  # [4, 32, 3, 32]  — каналы и высота поменялись
```

**Разница между view и reshape:** view требует, чтобы тензор был contiguous (данные лежат сплошным куском в памяти). После transpose() тензор перестаёт быть contiguous, и view упадёт с ошибкой. reshape в этом случае создаст копию. На практике для новичков лучше всегда использовать reshape.

```python
# squeeze — удаляет измерения размера 1
x = torch.randn(1, 3, 1, 32)
print(x.squeeze().shape)  # [3, 32] — убрал все единичные измерения
print(x.squeeze(dim=0).shape)  # [3, 1, 32] — убрал только первое

# unsqueeze — добавляет измерение размера 1
x = torch.randn(3, 32)
print(x.unsqueeze(0).shape)  # [1, 3, 32] — добавил батч-измерение в начало
print(x.unsqueeze(-1).shape) # [3, 32, 1] — добавил измерение в конец
```

### Индексация и срезы

Работает как в NumPy: квадратные скобки, срезы через `:`.

```python
x = torch.randn(4, 3, 32, 32)

# Первая картинка из батча
img = x[0]            # [3, 32, 32]

# Первые 2 картинки
batch_2 = x[:2]       # [2, 3, 32, 32]

# Красный канал первой картинки
red_channel = x[0, 0]  # [32, 32]

# Правый верхний угол 8×8 первой картинки
patch = x[0, :, :8, 24:]  # [3, 8, 8]

# Индексация по маске — выбираем элементы по условию
mask = x > 0
positive_values = x[mask]  # 1D тензор со всеми положительными числами

# Индексация списком индексов
indices = torch.tensor([0, 2, 3])
selected = x[indices]  # берём 0, 2 и 3 картинку из батча
```

### Математические операции

Поэлементные операции работают так же, как в NumPy. Есть две формы записи: через операторы (`+`, `-`, `*`) и через функции (`torch.add()`, `torch.mul()`):

```python
a = torch.tensor([1.0, 2.0, 3.0])
b = torch.tensor([4.0, 5.0, 6.0])

# Поэлементные (broadcasting работает как в NumPy)
print(a + b)    # [5, 7, 9]
print(a * b)    # [4, 10, 18]
print(a ** 2)   # [1, 4, 9]
print(a.sin())  # синус каждого элемента
print(a.clamp(1.5, 2.5))  # ограничить значения: [1.5, 2.0, 2.5]

# Матричное умножение — принципиально другая операция!
A = torch.randn(3, 4)
B = torch.randn(4, 5)
C = A @ B           # оператор @ — матричное умножение
C = torch.matmul(A, B)  # то же самое
print(C.shape)  # [3, 5]

# Для батчей матричное умножение тоже работает
batch_A = torch.randn(10, 3, 4)  # 10 матриц 3×4
batch_B = torch.randn(10, 4, 5)  # 10 матриц 4×5
batch_C = batch_A @ batch_B      # 10 результатов 3×5
```

**Важно:** `*` — это поэлементное умножение, а `@` — матричное. Смешивать их — частая ошибка.

### Агрегации (сумма, среднее, min, max)

Эти операции "схлопывают" тензор по указанному измерению:

```python
x = torch.randn(3, 4)

print(x.sum())           # скаляр — сумма всех элементов
print(x.mean())          # скаляр — среднее всех элементов
print(x.max())           # скаляр — максимальный элемент

# По конкретной оси
print(x.sum(dim=0))      # [4] — сумма по строкам (для каждой колонки)
print(x.sum(dim=1))      # [3] — сумма по колонкам (для каждой строки)

# Сохранить размерность — полезно для broadcasting
print(x.sum(dim=1, keepdim=True))  # [3, 1] — не схлопывает до 1D

# argmax — индекс максимального элемента
print(x.argmax())        # индекс по всему тензору
print(x.argmax(dim=1))   # индекс максимума в каждой строке
```

### Broadcasting — автоматическое расширение размерностей

Это механизм, который позволяет работать с тензорами разной формы, автоматически расширяя меньший до размера большего:

```python
a = torch.randn(3, 4)     # матрица 3×4
b = torch.tensor([1, 2, 3, 4])  # вектор длины 4

c = a + b  # b автоматически "растягивается" на 3 строки
# a:     [3, 4]
# b:     [4]     -> [1, 4] -> [3, 4] (broadcasting)
# res:   [3, 4]

# Правила broadcasting:
# 1. Если размерностей не хватает — добавляем 1 слева
# 2. Сравниваем каждую размерность: если 1 — растягиваем, если равны — ок
# 3. Иначе — ошибка

a = torch.randn(3, 1, 5)   # [3, 1, 5]
b = torch.randn(1, 4, 5)   # [1, 4, 5]
c = a + b                   # [3, 4, 5] — обе размерности растянуты
```

### Дополнительные полезные методы

```python
x = torch.randn(3, 4)

# Конкатенация — склеивание тензоров
a = torch.randn(3, 2)
b = torch.randn(3, 3)
c = torch.cat([a, b], dim=1)   # [3, 5] — склеили по колонкам

# Stack — склеивание с созданием нового измерения
a = torch.randn(3, 4)
b = torch.randn(3, 4)
c = torch.stack([a, b], dim=0)  # [2, 3, 4] — новый слой сверху

# where — выбор по условию
condition = torch.tensor([True, False, True])
result = torch.where(condition, a, b)  # если True — берём из a, иначе из b

# gather — сбор значений по индексам (удобно для NLP)
scores = torch.randn(3, 5)
indices = torch.tensor([[0], [2], [4]])
selected = scores.gather(1, indices)  # берём значения по индексам

# unique — уникальные значения
labels = torch.tensor([0, 0, 1, 2, 1, 0])
unique_labels = labels.unique()  # [0, 1, 2]
```

### GPU: перенос тензоров

```python
# Проверка: есть ли CUDA-совместимая видеокарта
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
print(f"Использую {device}")

# Создаём тензор сразу на нужном устройстве
x = torch.randn(100, 100, device=device)

# Или переносим существующий
y = torch.randn(100, 100)
y = y.to(device)       # универсальный способ
z = x + y              # теперь оба на одном устройстве

# Забрать обратно на CPU

```

**Почему устройство так важно?** Если вы забудете перенести тензор на GPU, а модель уже там — получите ошибку "Expected all tensors to be on the same device". Всегда следите, чтобы данные были на том же устройстве, что и модель.

### Тензоры и NumPy

```python
# numpy() — конвертирует в numpy (делит память, изменения отражаются в обоих)
x = torch.randn(3, 4)
np_arr = x.numpy()
np_arr[0, 0] = 99
print(x[0, 0].item())  # 99 — изменилось и в тензоре!

# from_numpy — обратная конвертация (тоже делит память)
np_arr2 = np.array([1, 2, 3])
t = torch.from_numpy(np_arr2)
t[0] = 100
print(np_arr2[0])  # 100 — изменилось везде
```

**Предупреждение:** общие данные — это удобно (не копируется лишний раз), но опасно. Если вы в одном месте измените значение, оно изменится и в другом. Если нужно независимое копирование — используйте `.clone()`.

## 2. Autograd — градиенты автоматически

Это магия, на которой держится весь PyTorch. Когда вы делаете операции над тензором, PyTorch запоминает, что и как вы делали (строит **граф вычислений**). Потом одной командой можно посчитать производную по любой переменной.

Зачем? При обучении нейронной сети нам нужно считать, как изменился loss (ошибка) при изменении каждого веса. Это и есть градиент. Без autograd пришлось бы прописывать формулы производных вручную — для сетей с миллионами параметров это нереально.

```python
# requires_grad=True — говорим PyTorch: "следи за этим тензором"
x = torch.tensor([2.0], requires_grad=True)

# Проводим вычисления — PyTorch запоминает весь путь
y = x ** 2 + 3 * x + 1   # y = 2² + 3*2 + 1 = 11

# backward() — считает производную dy/dx в точке x=2
y.backward()

# смотрим результат: dy/dx = 2x + 3, при x=2 это 7
print(x.grad)  # tensor([7.])
```

**Как это связано с нейронками?** Представьте, что `x` — это вес нейрона, `y` — ошибка модели. `x.grad` показывает, в какую сторону и насколько нужно изменить вес, чтобы ошибка уменьшилась. Делаем `x = x - lr * x.grad` — это и есть шаг обучения.

```python
# Важный нюанс: градиенты накапливаются!
# После каждого backward() градиент добавляется к предыдущему.
# Поэтому перед новым backward() нужно обнулять:
x.grad.zero_()
```

На практике вы вручную не будете работать с `requires_grad` на каждом тензоре — это происходит само, когда вы создаёте модель. Но понимать принцип нужно.

## 3. Создаём свой датасет (Dataset + DataLoader)

В реальной жизни данные — это не тензоры в памяти, а файлы на диске: картинки в папках, CSV-файлы, тексты. PyTorch предлагает стандартный способ организации данных через два класса, которые работают в паре:

**`Dataset`** — описывает, как достать один элемент данных. Вы пишете свой класс, наследующийся от Dataset, и объясняете PyTorch: "вот как открыть файл, вот как его преобразовать в тензор, вот какой ответ правильный". Dataset может быть любым — папка с картинками, CSV-файл, база данных, API.

**`DataLoader`** — берёт ваш Dataset и автоматически делает с ним удобные вещи: собирает элементы в батчи заданного размера, перемешивает их перед каждой эпохой, может загружать данные в несколько потоков параллельно (чтобы не ждать, пока откроется файл).

### Как устроен Dataset изнутри

У Dataset есть три метода, которые нужно реализовать:

- `__init__(self)` — загружает список всех файлов или готовит данные. Выполняется один раз при создании датасета.
- `__len__(self)` — возвращает количество элементов. Нужен DataLoader'у, чтобы знать, сколько батчей получится.
- `__getitem__(self, idx)` — принимает индекс и возвращает один элемент данных и его метку. Вот тут происходит самое интересное: чтение файла, преобразование в тензор, аугментация.

Давайте разберём на простом примере. Допустим, у нас есть зависимость y = 3x + шум. Мы хотим, чтобы модель научилась предсказывать y по x. Данных у нас нет — мы их сгенерируем:

```python
from torch.utils.data import Dataset, DataLoader

class MyDataset(Dataset):
    """Свой датасет: генерируем точки на прямой y = 3x + шум"""

    def __init__(self, n_samples=100):
        # __init__ — тут мы один раз создаём данные
        # torch.randn(n_samples, 1) даёт матрицу [n_samples, 1]
        # где каждое число — из нормального распределения (среднее 0, дисперсия 1)
        self.x = torch.randn(n_samples, 1)

        # Добавляем шум, чтобы было похоже на реальные данные
        # (в реальной жизни данные никогда не лежат на идеальной прямой)
        noise = torch.randn(n_samples, 1) * 0.1
        self.y = 3 * self.x + noise

    def __len__(self):
        # DataLoader вызывает len(dataset) чтобы понять, сколько всего примеров
        return len(self.x)

    def __getitem__(self, idx):
        # DataLoader вызывает dataset[i] для каждого i и собирает результат в батч
        # Возвращаем пару: (признак, целевое значение)
        return self.x[idx], self.y[idx]

# Создаём датасет на 200 точек
dataset = MyDataset(200)
print(f"Всего примеров: {len(dataset)}")  # 200
```

**Важно:** в `__getitem__` нельзя делать дорогие операции, которые повторяются для каждого вызова. Открытие файла, чтение с диска, декодирование — это нормально. А вот пересчёт сложных индексов, доступ к базе данных — лучше сделать один раз в `__init__`.

### DataLoader — упаковщик в батчи

Теперь нам нужно подавать данные модели не по одному, а батчами. За это отвечает DataLoader:

```python
# DataLoader заворачивает Dataset и добавляет удобства
dataloader = DataLoader(
    dataset,
    batch_size=16,    # сколько примеров в одном батче
    shuffle=True,     # перемешивать ли данные каждую эпоху
    num_workers=0,    # сколько процессов для загрузки (0 = без параллельности)
    drop_last=False   # выкинуть последний неполный батч?
)
```

**Параметры DataLoader подробно:**

- **batch_size** — самый важный параметр. Чем больше батч, тем стабильнее градиент, но тем больше нужно памяти GPU. 16-64 для маленьких сеток, 128-512 для больших.
- **shuffle=True** — обязательно перемешивайте данные перед каждой эпохой! Иначе модель может запомнить порядок, а не реальную зависимость.
- **num_workers=0** — сколько процессов использовать для загрузки. На Windows с num_workers > 0 бывают проблемы. 0 — работает всегда, 2-4 — быстрее, если данные тяжёлые (картинки).
- **drop_last=False** — если последний батч получился неполным (например, 200 / 16 = 12.5 → 12 полных + 1 неполный), выкинуть его. Иногда неполный батч ломает batch normalization.

Теперь посмотрим, что получается на выходе из DataLoader:

```python
# Итерируемся по DataLoader — он сам собирает элементы в батчи
for batch_x, batch_y in dataloader:
    print(f"Форма батча x: {batch_x.shape}")   # [16, 1] — 16 примеров, 1 признак
    print(f"Форма батча y: {batch_y.shape}")   # [16, 1] — 16 ответов
    print(f"Первое значение x в батче: {batch_x[0].item():.3f}")
    print(f"Соответствующий y: {batch_y[0].item():.3f} (должен быть ~3 * x)")
    break  # посмотрели первый батч — хватит
```

**Что делает DataLoader внутри?** Примерно так:

```python
# Упрощённая логика DataLoader:
indices = list(range(len(dataset)))  # [0, 1, 2, ..., 199]
if shuffle:
    random.shuffle(indices)          # перемешиваем

for i in range(0, len(indices), batch_size):  # шагаем по 16
    batch_indices = indices[i:i+batch_size]
    batch_x = torch.stack([dataset[idx][0] for idx in batch_indices])
    batch_y = torch.stack([dataset[idx][1] for idx in batch_indices])
    yield batch_x, batch_y
```

То есть DataLoader просто вызывает `dataset[i]` для каждого индекса в батче и склеивает результаты в один тензор с помощью stack.

        x = torch.tensor(self.X[idx])
        y = torch.tensor(self.Y[idx])
        return x, y

### Пример с картинками: папка с файлами

А вот как выглядит датасет для картинок. Здесь уже нужно читать файлы с диска и применять преобразования:

```python
from PIL import Image
from torchvision import transforms

class ImageDataset(Dataset):
    """Датасет из папки с картинками.
    Структура: data/ — все .jpg файлы,
    labels.csv — файл с названием файла и меткой класса."""

    def __init__(self, img_dir, csv_path, transform=None):
        self.img_dir = img_dir
        self.df = pd.read_csv(csv_path)  # колонки: filename, label
        self.transform = transform or transforms.ToTensor()

    def __len__(self):
        return len(self.df)

    def __getitem__(self, idx):
        # Берём имя файла и метку из CSV
        img_name = self.df.iloc[idx, 0]
        label = self.df.iloc[idx, 1]

        # Открываем картинку
        img_path = f"{self.img_dir}/{img_name}"
        image = Image.open(img_path).convert('RGB')  # все картинки в RGB

        # Применяем преобразования (нормализация, ресайз и т.д.)
        # transforms.ToTensor() конвертирует PIL Image в тензор [C, H, W]
        image = self.transform(image)

        return image, label

# transforms — это цепочка преобразований, которые применяются к картинке
transform = transforms.Compose([
    transforms.Resize((224, 224)),        # меняем размер
    transforms.RandomHorizontalFlip(),    # случайно отражаем (аугментация)
    transforms.ToTensor(),                # PIL -> тензор [0, 1]
    transforms.Normalize(                 # нормализация под предобученную модель
        mean=[0.485, 0.456, 0.406],
        std=[0.229, 0.224, 0.225]
    )
])

# dataset = ImageDataset("data/train/", "data/labels.csv", transform=transform)
```

**Что такое transforms и зачем они нужны?** transforms — это набор преобразований для данных. Они решают две задачи:
1. **Привести данные к единому формату** — все картинки должны быть одного размера, одного типа (RGB), одного диапазона значений.
2. **Аугментация** — искусственно увеличиваем количество данных, случайно меняя картинки (повороты, отражения, шум). Модель учится быть устойчивой к этим изменениям.

### Свой Dataset с аугментацией

Можно добавить аугментацию прямо в `__getitem__` без torchvision:

```python
class MyAugmentedDataset(Dataset):
    def __init__(self, n_samples=1000):
        self.x = torch.randn(n_samples, 1)
        noise = torch.randn(n_samples, 1) * 0.1
        self.y = 3 * self.x + noise

    def __len__(self):
        return len(self.x)

    def __getitem__(self, idx):
        x = self.x[idx]
        y = self.y[idx]

        # Аугментация: добавляем шум прямо во время загрузки
        # Каждый раз, когда DataLoader запрашивает элемент,
        # мы добавляем случайный шум — модель видит разные данные
        if self.augment:
            x = x + torch.randn(1) * 0.05

        return x, y
```

**Почему аугментация делается в `__getitem__`, а не в `__init__`?** Потому что аугментация должна быть случайной каждый раз, когда модель видит данные. Если сделать её в `__init__`, то на каждой эпохе данные будут одинаковыми, и смысла в аугментации не будет.

### Dataset из словаря — когда данные уже в памяти

Иногда данные уже загружены (например, из pickle или h5). Тогда датасет делается максимально просто:

```python
class DictDataset(Dataset):
    """Универсальный датасет из готовых тензоров."""

    def __init__(self, features, labels):
        # features: torch.Tensor формы [N, num_features]
        # labels: torch.Tensor формы [N] или [N, 1]
        self.features = features
        self.labels = labels

        # Проверка, что количество примеров совпадает
        assert len(features) == len(labels), "features и labels разной длины!"

    def __len__(self):
        return len(self.features)

    def __getitem__(self, idx):
        return self.features[idx], self.labels[idx]

# Использование:
# X = torch.randn(1000, 10)  # 1000 примеров, 10 признаков
# y = torch.randint(0, 2, (1000,))  # 1000 меток (0 или 1)
# dataset = DictDataset(X, y)
```

### Итог по Dataset

1. **Dataset** — это класс с тремя методами: `__init__`, `__len__`, `__getitem__`.
2. `__init__` — подготавливаем список того, что будем читать (пути к файлам, индексы).
3. `__getitem__` — читаем один элемент по индексу, преобразуем в тензор, возвращаем (данные, метка).
4. **DataLoader** — берёт Dataset, разбивает на батчи, перемешивает, загружает параллельно.
5. **Аугментация** делается в `__getitem__`, чтобы каждый раз получать разные данные.
6. **transform** из torchvision — удобный способ навесить цепочку преобразований на картинки.

## 4. Создаём модель (Linear -> ReLU -> Linear)

Теперь самое интересное — нейронная сеть. В PyTorch модель — это класс, который наследуется от `nn.Module`. В `__init__` вы объявляете слои (параметры модели), а в `forward` описываете, как данные проходят через эти слои.

```python
import torch.nn as nn

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()  # вызываем родительский __init__ (обязательно)

        # nn.Linear(in_features, out_features) — полносвязный (линейный) слой
        # Он делает: output = input @ W + b, где W — веса, b — смещение
        # in_features=1 — на вход приходит 1 число (наше x)
        # out_features=16 — внутри слой делает 16 разных преобразований
        self.fc1 = nn.Linear(1, 16)

        # ReLU — функция активации. Она делает все отрицательные числа нулями
        # Зачем? Если не ставить активацию, то два линейных слоя подряд
        # можно схлопнуть в один — сеть не сможет изучать сложные зависимости
        self.relu = nn.ReLU()

        # Второй линейный слой: 16 нейронов -> 1 выход (наше предсказанное y)
        self.fc2 = nn.Linear(16, 1)

    def forward(self, x):
        # forward — описывает, как данные идут по сети
        # x приходит форме [batch_size, 1]

        x = self.fc1(x)   # [batch_size, 16] — 16 признаков
        x = self.relu(x)  # [batch_size, 16] — убрали отрицательные
        x = self.fc2(x)   # [batch_size, 1] — одно выходное число

        return x

model = MyModel()
print(model)
# MyModel(
#   (fc1): Linear(in_features=1, out_features=16, bias=True)
#   (relu): ReLU()
#   (fc2): Linear(in_features=16, out_features=1, bias=True)
# )
```

**Почему именно Linear -> ReLU -> Linear?** Это простейшая нейронная сеть с одним скрытым слоем. Первый Linear расширяет размерность (1 → 16), ReLU добавляет нелинейность, второй Linear сжимает обратно (16 → 1). Теоретически такая сеть может аппроксимировать любую функцию (теорема об универсальной аппроксимации).

**А что такое параметры модели?** У каждого Linear слоя есть веса (`weight`) и смещения (`bias`). В самом начале они случайные. В процессе обучения мы их будем менять:

```python
# Посмотреть все параметры модели
for name, param in model.named_parameters():
    print(f"{name}: {param.shape}")
# fc1.weight: torch.Size([16, 1])
# fc1.bias: torch.Size([16])
# fc2.weight: torch.Size([1, 16])
# fc2.bias: torch.Size([1])

# Всего параметров: 16*1 + 16 + 1*16 + 1 = 49 — крошечная сетка
```

## 5. Задаём функцию потерь и оптимизатор

Модель есть, данные есть. Но чтобы обучаться, нужно две вещи:

**Функция потерь (loss function)** — численная оценка того, насколько модель ошибается. Мы будем её минимизировать. Чем меньше loss, тем лучше предсказания.

**Оптимизатор (optimizer)** — алгоритм, который меняет веса модели так, чтобы уменьшить loss. Самый простой — SGD (Stochastic Gradient Descent).

```python
# MSE (Mean Squared Error) — средний квадрат ошибки
# Используется для регрессии (предсказание числа)
# Формула: loss = mean((y_pred - y_true)²)
# Почему квадрат? Чтобы большие ошибки наказывались сильнее
criterion = nn.MSELoss()

# SGD — Stochastic Gradient Descent (стохастический градиентный спуск)
# model.parameters() — все веса модели, которые будем менять
# lr (learning rate) — скорость обучения, 0.01
# Если lr слишком большой — loss будет скакать и не сойдётся
# Если lr слишком маленький — обучение будет очень медленным
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
```

**Как они работают вместе?**
1. Модель предсказывает → получаем loss
2. Считаем градиент loss по каждому параметру (backward)
3. Оптимизатор сдвигает параметры в сторону уменьшения loss
4. Повторяем

## 6. Обучаем модель

Теперь соединяем всё вместе. У нас есть данные (dataloader), модель (model), функция потерь (criterion) и оптимизатор (optimizer). Запускаем цикл обучения.

Одна полная эпоха — это когда модель прошла по всем данным (всем батчам) один раз. Обычно нужно много эпох, чтобы loss снизился до приемлемого уровня.

```python
epochs = 100  # сколько раз пройдём по всем данным

for epoch in range(epochs):
    # Проходим по всем батчам
    for batch_x, batch_y in dataloader:
        # ---- Шаг 1: Предсказание (forward pass) ----
        # Пропускаем батч через модель
        # model() автоматически вызывает forward()
        pred = model(batch_x)
        # pred — это тензор формы [batch_size, 1], предсказанные y

        # ---- Шаг 2: Считаем ошибку (loss) ----
        # Сравниваем предсказание с правильным ответом
        loss = criterion(pred, batch_y)
        # loss — это скаляр (одно число)

        # ---- Шаг 3: Обнуляем градиенты ----
        optimizer.zero_grad()
        # Это важно! Градиенты накапливаются по умолчанию (см. раздел про autograd)
        # Если не обнулить, на следующем шаге градиенты сложатся со старыми

        # ---- Шаг 4: Считаем градиенты (backward pass) ----
        loss.backward()
        # Теперь у каждого параметра модели в .grad лежит производная loss
        # по этому параметру

        # ---- Шаг 5: Обновляем веса ----
        optimizer.step()
        # SGD делает: param = param - lr * param.grad для каждого параметра
        # Это один шаг градиентного спуска

    # Каждые 20 эпох печатаем текущий loss
    if (epoch + 1) % 20 == 0:
        # loss.item() — достаём число из тензора
        print(f"Эпоха {epoch+1}/{epochs}, Loss: {loss.item():.4f}")
```

**Как понять, что обучение идёт правильно?** Loss должен уменьшаться с каждой эпохой. Если loss растёт — lr слишком большой. Если почти не меняется — lr слишком маленький или модель слабая. В нашем случае loss будет падать примерно до 0.01 (это уровень шума, который мы добавили в данные).

**Почему каждый шаг называется эпохой и зачем их много?** Один проход по всем данным — это эпоха. Одной эпохи обычно мало — модель делает только один шаг на каждом батче, и веса меняются незначительно. Нужно много эпох, чтобы веса постепенно пришли к оптимальным значениям.

## Структура работы в PyTorch (шпаргалка)

Весь пайплайн укладывается в 5 шагов:

```
1. ДАННЫЕ
   Dataset → __init__() / __len__() / __getitem__()
   DataLoader → разбивает на батчи, перемешивает

2. МОДЕЛЬ
   class Net(nn.Module) → __init__() объявляем слои
                        → forward() описываем проход данных

3. ФУНКЦИЯ ПОТЕРЬ
   nn.MSELoss() — для чисел (регрессия)
   nn.CrossEntropyLoss() — для классов (классификация)

4. ОПТИМИЗАТОР
   optim.SGD() — простой и надёжный
   optim.Adam() — быстрее сходится, популярен на практике

5. ЦИКЛ ОБУЧЕНИЯ
   for epoch in range(epochs):
       for x, y in dataloader:
           pred = model(x)          # forward
           loss = criterion(pred, y)  # считаем ошибку
           optimizer.zero_grad()    # обнуляем градиенты
           loss.backward()          # считаем градиенты
           optimizer.step()         # обновляем веса
```

- CNN (сверточные слои) — для картинок (torch.nn.Conv2d)
- RNN / LSTM — для последовательностей (текст, временные ряды)
- Датасеты из файлов — ImageFolder для картинок, CSV для таблиц
- Трансферное обучение — взять готовую модель и дообучить под свою задачу
