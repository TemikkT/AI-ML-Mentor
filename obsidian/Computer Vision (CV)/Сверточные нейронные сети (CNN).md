# Свёрточные нейронные сети (CNN) — полная база

## Почему обычная нейронная сеть не подходит для картинок

Представьте картинку 256×256 пикселей, цветная (RGB — 3 канала). Если подать её на вход обычной полносвязной сети (Linear layer), то каждый пиксель — это один признак. Получаем 256 × 256 × 3 = 196 608 входов.

Если первый скрытый слой имеет 1000 нейронов, то количество весов: 196 608 × 1000 = **196 миллионов параметров**. Это:
1. Не влезет в память
2. Переобучится на маленьком датасете
3. Будет считать столетиями

Но главная проблема не в размере. Полносвязный слой не знает, что пиксели рядом — это часть одного объекта, а пиксели на другой стороне картинки — совсем другое. Он относится к каждому пикселю как к независимому признаку. Это неправильно — у картинок есть **пространственная структура**.

**CNN решает обе проблемы:** использует локальность (смотрит на маленькие участки) и разделение весов (один и тот же фильтр применяется ко всей картинке).

---

## Принцип работы свёртки

### Идея: скользящее окно с фильтром

Свёртка работает просто: берём маленькую матрицу — **фильтр (ядро, kernel)**, обычно 3×3 или 5×5. И скользим этим фильтром по всей картинке. На каждой позиции умножаем значения фильтра на значения пикселей под ним и складываем. Получается одно число.

После прохода всей картинки получается новая матрица — **карта признаков (feature map)**.

```
Картинка 5×5          Фильтр 3×3         Результат 3×3
[1 2 3 4 5]          [1 0 1]            [? ? ?]
[6 7 8 9 0]          [0 1 0]    →       [? ? ?]
[1 2 3 4 5]          [1 0 1]            [? ? ?]
[6 7 8 9 0]
[1 2 3 4 5]

Первый шаг: фильтр накрывает левый верхний угол 3×3
[1 2 3]           [1 0 1]
[6 7 8]    ×      [0 1 0]    = 1*1 + 2*0 + 3*1 + 6*0 + 7*1 + 8*0 + 1*1 + 2*0 + 3*1
[1 2 3]           [1 0 1]    = 1 + 3 + 7 + 1 + 3 = 15
```

**Что делает фильтр?** Он ищет определённый паттерн. Один фильтр ищет горизонтальные границы, другой — вертикальные, третий — углы, четвёртый — текстуры.

```python
import torch
import torch.nn as nn

# Простейший свёрточный слой
conv = nn.Conv2d(
    in_channels=3,    # цветов RGB — 3 канала на входе
    out_channels=16,  # сколько фильтров (карт признаков)
    kernel_size=3,    # размер фильтра 3×3
    stride=1,         # шаг скольжения (1 — через каждый пиксель)
    padding=1         # добавляем рамку из нулей, чтобы размер не уменьшался
)

# Вход: батч из 8 картинок, RGB 64×64
x = torch.randn(8, 3, 64, 64)  # [batch, channels, height, width]
out = conv(x)
print(out.shape)  # [8, 16, 64, 64] — 16 карт признаков, размер 64×64
```

**Что изменилось:** было 3 канала (RGB), стало 16 признаков. Размер картинки не изменился (спасибо padding). Каждый из 16 выходных каналов — это результат работы одного фильтра.

### Почему это работает?

Фильтр 3×3 видит только локальную область — 9 пикселей. Он ищет простой паттерн: границу, уголок, точку. Но на следующем слое CNN фильтр видит уже не пиксели, а результаты работы предыдущих фильтров — то есть комбинации простых паттернов. Так сеть строит иерархию:

- **Первый слой:** линии, границы, простые текстуры
- **Второй слой:** комбинации линий — углы, кружки, дуги
- **Третий слой:** части объектов — глаза, колёса, окна
- **Глубокие слои:** целые объекты — лица, машины, здания

Это называется **иерархия признаков**. Чем глубже — тем более абстрактные и сложные вещи видит сеть. Именно это делает CNN такими эффективными для изображений.

## Padding — паддинг

Когда фильтр 3×3 скользит по картинке 5×5, на каждом шаге фильтр на один элемент смещается. На выходе получится карта 3×3 (5-3+1 = 3). Картинка сжимается. Если добавить несколько слоёв, картинка быстро уменьшится до 1×1 и мы потеряем информацию.

**Решение: padding** — добавляем рамку из нулей (или других значений) вокруг картинки.

```python
# Без padding — размер уменьшается
conv_no_pad = nn.Conv2d(3, 16, kernel_size=3, padding=0)
out_no_pad = conv_no_pad(torch.randn(1, 3, 64, 64))
print(out_no_pad.shape)  # [1, 16, 62, 62] — уменьшилось на 2

# С padding=1 — размер сохраняется
conv_pad = nn.Conv2d(3, 16, kernel_size=3, padding=1)
out_pad = conv_pad(torch.randn(1, 3, 64, 64))
print(out_pad.shape)  # [1, 16, 64, 64] — размер тот же

# padding = kernel_size // 2 — формула для сохранения размера
```

**Зачем padding?** Без него каждый свёрточный слой уменьшает картинку на (kernel_size - 1). При глубине сети в 20 слоёв с kernel=3 картинка 256×256 превратится в 256 - 2*20 = 216×216 — не критично. Но с kernel=7 уже 256 - 6*20 = 136×136 — ощутимо. Padding позволяет строить сколь угодно глубокие сети без потери пространственного разрешения.

## Stride — шаг свёртки

Stride (шаг) — на сколько пикселей сдвигается фильтр на каждом шаге.

```python
# stride=1 — стандартный, каждый пиксель, размер почти не меняется
conv1 = nn.Conv2d(3, 16, kernel_size=3, stride=1, padding=1)
# out: [batch, 16, H, W] — тот же размер

# stride=2 — пропускаем каждый второй, уменьшаем в ~2 раза
conv2 = nn.Conv2d(3, 16, kernel_size=3, stride=2, padding=1)
# out: [batch, 16, H/2, W/2]

# Иногда stride=2 используют вместо Pooling — это "свёрточный downsampling"
```

**Stride = 2 + kernel = 3 + padding = 1** — популярная комбинация, которая уменьшает размер вдвое. Так можно постепенно уменьшать картинку без отдельного слоя пулинга.

---

## Каналы и глубина

На вход первого слоя приходит цветная картинка — 3 канала (RGB). На выходе — 16, 32, 64 канала (это out_channels). Что это значит?

Каждый выходной канал — это результат работы **своего фильтра**. Первый фильтр ищет горизонтальные грани, второй — вертикальные, третий — красные точки и т.д.

Но когда мы идём глубже, каждый новый фильтр смотрит не на 3 канала (RGB), а на 16, 32 или 64 — на выходы предыдущих слоёв. Он учится комбинировать признаки.

```python
# Типичный блок CNN
self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)   # 3 → 32
self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)  # 32 → 64
self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1) # 64 → 128

# Каждый слой:
# - Принимает in_channels (число признаков с предыдущего слоя)
# - Выдаёт out_channels (свои признаки)
# - Пространственный размер (H, W) постепенно уменьшается через stride или pooling
# - Глубина (количество каналов) растёт: 3 → 32 → 64 → 128
```

**Почему глубина растёт, а размер падает?** Потому что первые слои ищут простые паттерны (их мало — 16-32), а глубокие слои ищут сложные комбинации (их много — 128-512). При этом пространственный размер уменьшается, чтобы не взорвать количество вычислений.

## ReLU — нелинейность после свёртки

Свёртка — это линейная операция (умножение + сложение). Без нелинейности сколько слоёв ни добавь — всё схлопнется в одно линейное преобразование (как умножение чисел: 2*(3*x) = (2*3)*x = 6*x).

**ReLU** — самая популярная функция активации в CNN. Она делает все отрицательные значения нулями:

```python
relu = nn.ReLU()
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])
print(relu(x))  # [0.0, 0.0, 0.0, 1.0, 2.0]
```

**Почему именно ReLU, а не sigmoid/tanh?**
1. Sigmoid насыщается при больших значениях (градиент ≈ 0) — обучение останавливается. У ReLU нет этой проблемы.
2. ReLU просто обнуляет отрицательные — это очень дёшево вычислительно.
3. На практике ReLU сходится быстрее.

```python
# Типичный блок: Conv → ReLU
x = torch.randn(8, 3, 64, 64)
x = conv(x)     # свертка
x = relu(x)     # нелинейность
# Если этого не сделать — три слоя Conv подряд
# можно заменить одним (линейная комбинация линейных = линейная)
```

## MaxPooling — уменьшение размерности с умом

После свёртки у нас есть карты признаков. Но они всё ещё большого размера (64×64, 32×32). Нужно уменьшать, чтобы:
1. Меньше вычислений на следующих слоях
2. Меньше параметров → меньше переобучения
3. Модель училась более абстрактным признакам

**MaxPooling** — берёт окошко (обычно 2×2) и оставляет только максимальное значение. Шаг обычно равен размеру окна, так что размер уменьшается вдвое.

```python
pool = nn.MaxPool2d(kernel_size=2, stride=2)

x = torch.randn(1, 16, 64, 64)   # 16 карт 64×64
out = pool(x)
print(out.shape)  # [1, 16, 32, 32] — размер уменьшился вдвое
```

**Почему MaxPooling, а не AveragePooling?** MaxPooling оставляет самый сильный сигнал — самую яркую границу, самый контрастный уголок. Это помогает модели фокусироваться на важных деталях. AveragePooling (среднее по окну) сглаживает — используется реже, в основном в конце сети перед классификатором.

```python
# Можно использовать stride=2 в Conv2d вместо MaxPooling
# Современные сети (ResNet) предпочитают так — меньше информации теряется
conv_down = nn.Conv2d(16, 32, kernel_size=3, stride=2, padding=1)
```

**Что теряется при Pooling:** информация о том, ГДЕ именно был паттерн. Модели важно знать, что глаз есть, но неважно, что он на 12 пикселей левее. Это называется **трансляционная инвариантность** — модель не зависит от точного положения объекта.

## Полносвязный слой в конце — классификация

После нескольких блоков Conv → ReLU → Pooling мы получаем много карт признаков маленького размера (например, 128 каналов × 4×4). Теперь нужно сделать из этого предсказание (класс картинки).

Для этого "разворачиваем" все каналы в один длинный вектор и подаём на полносвязный слой:

```python
class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()

        # Feature extraction — извлекаем признаки
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.relu = nn.ReLU()
        self.pool = nn.MaxPool2d(2, 2)    # уменьшаем вдвое

        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        # после двух pool: 64×64 → 32×32 → 16×16

        # Классификатор — принимает вектор 64 * 16 * 16
        self.fc1 = nn.Linear(64 * 16 * 16, 128)
        self.fc2 = nn.Linear(128, num_classes)

    def forward(self, x):
        # x: [batch, 3, 64, 64]

        # Свёрточная часть
        x = self.pool(self.relu(self.conv1(x)))  # [batch, 32, 32, 32]
        x = self.pool(self.relu(self.conv2(x)))  # [batch, 64, 16, 16]

        # Разворачиваем в вектор для полносвязных слоёв
        x = x.view(x.size(0), -1)  # [batch, 64 * 16 * 16]

        # Классификация
        x = self.relu(self.fc1(x))  # [batch, 128]
        x = self.fc2(x)             # [batch, num_classes]

        return x
```

**Что такое view?** `x.view(batch_size, -1)` разворачивает каждый пример в одномерный вектор. -1 означает "подбери размер автоматически": было [64, 64, 16, 16] → стало [64, 16384]. После этого можно подавать на Linear слои.

## Adaptive Pooling — решение проблемы с размерами

Проблема: если картинки разного размера, то после свёрток размер feature map будет разным, и мы не сможем подать их на полносвязный слой (он ждёт фиксированный вход).

**Adaptive Pooling** — пулинг, который всегда выдаёт заданный выходной размер, независимо от входного:

```python
# Всегда выдаёт 1×1 независимо от входа
pool_global = nn.AdaptiveAvgPool2d((1, 1))

x_small = torch.randn(1, 64, 8, 8)
x_large = torch.randn(1, 64, 16, 16)

print(pool_global(x_small).shape)   # [1, 64, 1, 1]
print(pool_global(x_large).shape)   # [1, 64, 1, 1]
```

Теперь можно убрать view и сделать классификатор с фиксированным входом:

```python
class FlexibleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),
            nn.Conv2d(32, 64, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),
        )
        # Adaptive Pooling — всегда 64 канала × 1×1
        self.pool = nn.AdaptiveAvgPool2d((1, 1))
        self.classifier = nn.Linear(64, num_classes)

    def forward(self, x):
        x = self.features(x)
        x = self.pool(x)          # [batch, 64, 1, 1]
        x = x.view(x.size(0), -1) # [batch, 64]
        x = self.classifier(x)    # [batch, num_classes]
        return x
```

## Размеры входов и выходов: формула

Чтобы рассчитать размер выхода после свёртки:

```
out_size = (in_size + 2 * padding - kernel_size) // stride + 1
```

```python
def conv_output_size(in_size, kernel_size, padding=0, stride=1):
    return (in_size + 2 * padding - kernel_size) // stride + 1

print(conv_output_size(64, 3, 1, 1))  # 64 — с padding=1 размер сохраняется
print(conv_output_size(64, 3, 0, 1))  # 62 — без padding уменьшается на 2
print(conv_output_size(64, 3, 1, 2))  # 32 — с stride=2 уменьшается вдвое
```

## Количество параметров в свёртке

Сравните с полносвязным слоём. Conv2d с 3 входами и 16 выходами, kernel_size=3:

```python
# Пример: считаем параметры
conv = nn.Conv2d(3, 16, kernel_size=3)
total_params = sum(p.numel() for p in conv.parameters())
print(total_params)  # 448 = 3*16*3*3 + 16

# Сравните: полносвязный слой с 3*64*64 входами и 100 нейронами
fc = nn.Linear(3 * 64 * 64, 100)
fc_params = sum(p.numel() for p in fc.parameters())
print(fc_params)  # 1 228 900 — в 2700 раз больше!
```

**Почему так мало?** Каждый фильтр 3×3 применяется ко всем входным каналам. Один фильтр = 3×3×3 весов (каналы × ширина × высота) + 1 bias = 28 параметров. 16 фильтров = 28 × 16 = 448. И это работает со всей картинкой 64×64! В полносвязном слое для каждого пикселя свой вес — вот почему их на 3 порядка больше.

## Dropout — борьба с переобучением

Проблема: сеть может запомнить обучающие данные вместо того, чтобы выучить закономерности. 

**Dropout** — во время тренировки случайно "выключает" часть нейронов (обнуляет их). Это заставляет сеть не полагаться на один нейрон, а распределять знания между всеми.

```python
dropout = nn.Dropout(p=0.5)  # 50% нейронов выключится на каждом шаге

x = torch.randn(1, 10)
out_train = dropout(x)  # во время тренировки — половина нейронов в 0

dropout.eval()  # переключаем в режим оценки
out_eval = dropout(x)  # во время инференса Dropout не работает!
```

```python
# Dropout после полносвязных слоёв — обычная практика
class CNNWithDropout(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 32, 3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.fc1 = nn.Linear(64 * 16 * 16, 128)
        self.fc2 = nn.Linear(128, 10)
        self.dropout = nn.Dropout(0.5)

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x)))
        x = self.pool(F.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        x = F.relu(self.fc1(x))
        x = self.dropout(x)  # Dropout только между полносвязными слоями
        x = self.fc2(x)
        return x
```

**Важно:** Dropout применяется только во время обучения. PyTorch автоматически отключает его в `model.eval()`.

## Batch Normalization — стабилизация обучения

Проблема: значения в сети могут сильно меняться от слоя к слою. Начальные слои обучаются, их выходы меняются — следующие слои каждый раз видят данные с разным распределением. Это замедляет обучение.

**Batch Normalization** — нормализует выход каждого слоя внутри одного батча: вычитает среднее батча и делит на стандартное отклонение. Потом применяет обучаемые сдвиг и масштаб (гамма и бета).

```python
# BatchNorm2d — для свёрточных слоёв (каналы)
# BatchNorm1d — для полносвязных слоёв

class CNNWithBN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 32, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(32)    # нормализация после conv1
        self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(64)    # нормализация после conv2
        self.pool = nn.MaxPool2d(2, 2)
        self.fc = nn.Linear(64 * 16 * 16, 10)

    def forward(self, x):
        # Conv → BatchNorm → ReLU — стандартный порядок
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        x = x.view(x.size(0), -1)
        x = self.fc(x)
        return x
```

**Почему BatchNorm + ReLU в таком порядке?** Есть спор. Статья Ioffe & Szegedy предлагала `Conv → BN → ReLU`. На практике часто встречается `Conv → ReLU → BN`. Разница небольшая, работает и так, и так.

**Что даёт BatchNorm:**
1. Можно использовать больший learning rate (обучение стабильнее)
2. Меньше зависит от инициализации весов
3. Работает как регуляризация (добавляет шум через статистику батча)
4. Ускоряет сходимость в разы

**Важно:** на маленьких батчах (batch_size < 16) BN работает плохо — статистика батча слишком шумная.

## Собираем полноценную CNN

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    """Простая CNN для классификации картинок 64×64"""

    def __init__(self, num_classes=10):
        super().__init__()

        # Блок 1: 64×64 → 32×32
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.bn1 = nn.BatchNorm2d(32)
        self.pool = nn.MaxPool2d(2, 2)

        # Блок 2: 32×32 → 16×16
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.bn2 = nn.BatchNorm2d(64)

        # Блок 3: 16×16 → 8×8
        self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1)
        self.bn3 = nn.BatchNorm2d(128)

        # Классификатор
        # После 3-х MaxPooling: 64 → 32 → 16 → 8
        self.fc1 = nn.Linear(128 * 8 * 8, 256)
        self.dropout = nn.Dropout(0.5)
        self.fc2 = nn.Linear(256, num_classes)

    def forward(self, x):
        # Свёрточные блоки
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        x = self.pool(F.relu(self.bn3(self.conv3(x))))

        # Разворачиваем
        x = x.view(x.size(0), -1)

        # Классификация
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)

        return x

model = SimpleCNN(num_classes=10)

# Проверка: подали картинку 64×64
test_input = torch.randn(4, 3, 64, 64)
output = model(test_input)
print(output.shape)  # [4, 10] — 4 предсказания, 10 классов
```

## Структура CNN: шпаргалка

```
Вход: [batch, 3, H, W]

[Conv2d → BatchNorm → ReLU → MaxPool2d] × N блоков
  ↓
AdaptiveAvgPool2d(1) — глобальный пулинг
  ↓
Flatten — разворачиваем в вектор
  ↓
Linear → ReLU → Dropout
  ↓
Linear — выход (logits)
```

**Типичные размеры фильтров:**
- Kernel: чаще всего 3×3 (реже 5×5, 7×7 только в первом слое)
- Stride: 1 или 2
- Padding: kernel_size // 2 (чтобы сохранить размер)
- Out channels: 32 → 64 → 128 → 256 (удваиваем)

**Советы:**
1. Всегда используйте BatchNorm — ускоряет обучение в разы
2. Dropout ставьте перед последним полносвязным слоем (p=0.5)
3. MaxPooling после каждого-двух свёрточных слоёв
4. Начинайте с маленького количества фильтров (32) и увеличивайте
5. Если картинки маленькие (28×28), MaxPooling может быть только один раз
6. Для больших картинок (224×224) можно пулить 4-5 раз
