# Статистика для машинного обучения

## Генеральная совокупность и выборка

В статистике мы почти никогда не знаем правду про всех людей/объекты — это **генеральная совокупность** (population). Мы работаем с маленькой частью — **выборкой** (sample). По выборке мы пытаемся угадать, что происходит в совокупности.

Проблема: выборка может быть нерепрезентативной. Пример: хотим узнать средний рост в России, но опросили только баскетболистов. Выборка смещена.

**Случайная выборка** — каждый элемент совокупности имеет равные шансы попасть в выборку. Только на случайной выборке работают статистические методы.

---

## Выборочное среднее и дисперсия

Среднее выборки — это оценка среднего всей совокупности. Дисперсия — оценка разброса.

```python
import numpy as np

data = [4, 5, 6, 7, 8, 9, 10]

mean = np.mean(data)      # 7.0 — сумма / количество
variance = np.var(data, ddof=0)   # дисперсия по выборке (делим на n)
var_unbiased = np.var(data, ddof=1)  # несмещённая дисперсия (делим на n-1)
std = np.std(data, ddof=1)         # стандартное отклонение
```

**Почему ddof=1 (деление на n-1)?** Если мы берём выборку из совокупности и считаем дисперсию, деление на n даёт смещённую оценку (немного занижает реальный разброс). Деление на n-1 исправляет это. Чем больше выборка, тем меньше разница между n и n-1. На практике всегда используйте ddof=1 для выборок.

## Bias (смещение) и Variance (разброс)

Это фундаментальное понятие для ML. **Bias** — насколько модель в среднем ошибается. **Variance** — насколько сильно предсказания модели меняются при смене обучающих данных.

### Bias — систематическая ошибка

Bias — это когда модель последовательно ошибается в одну сторону. Представьте, что вы стреляете по мишени. Bias — это если все ваши выстрелы сгруппированы левее центра. Вы стабильно мажете влево.

В ML bias — это предположения, которые модель делает о данных:
- Линейная регрессия предполагает, что зависимость линейная. Если данные на самом деле квадратичные, модель будет ошибаться систематически — высокий bias.
- Высокий bias = модель слишком простая (underfitting). Она не улавливает закономерности в данных.

**Пример:** предсказываем цену дома только по площади (одна фича). Это модель с высоким bias — она упрощает реальность, игнорируя местоположение, этаж, состояние.

### Variance — чувствительность к данным

Variance — это насколько сильно изменится модель, если её переобучить на другом наборе данных (из той же совокупности). Если модель каждый раз даёт сильно разные результаты — у неё высокий variance.

Вернёмся к мишени. Высокий variance — это когда выстрелы разбросаны по всей мишени: один влево, второй вправо, третий вверх. Нет стабильности.

В ML высокий variance — это когда модель выучила не только закономерности, но и шум в данных (overfitting). Она идеально работает на обучающих данных, но проваливается на новых.

**Пример:** дерево решений глубиной 100 на 50 примерах. Оно запомнит каждый пример, но на новых данных будет ошибаться — высокий variance.

### Компромисс bias-variance

Между bias и variance всегда компромисс. Усложняете модель — bias падает (модель лучше ловит закономерности), но variance растёт (модель начинает ловить шум). Упрощаете модель — variance падает, но bias растёт.

```python
# Наглядный пример: полиномиальная регрессия разной степени
# степень 1 (линейная) — высокий bias, низкий variance
# степень 3 — умеренный bias, умеренный variance
# степень 15 — низкий bias, высокий variance (overfitting)

from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import make_pipeline

def fit_poly(degree, X, y):
    model = make_pipeline(
        PolynomialFeatures(degree),
        LinearRegression()
    )
    model.fit(X, y)
    return model

# Степень 1: недообучение (high bias)
# Степень 15: переобучение (high variance)
# Нужно найти золотую середину через кросс-валидацию
```

**Как это выглядит на графике ошибок:**

```
Ошибка
  ^
  |               полная ошибка (test error)
  |            ↗
  |         ↗
  | bias ↘ ↗ variance
  |       ↘
  |        ↘
  +------------------→ Сложность модели
     under    over
     fitting  fitting
```

- Слева (простая модель): ошибка высокая из-за bias
- Справа (сложная модель): ошибка высокая из-за variance
- Посередине — оптимальная сложность

### Как бороться

**С высоким bias (underfitting):**
- Добавить больше фич
- Усложнить модель (нейронная сеть вместо линейной регрессии, больше слоёв/деревьев)
- Уменьшить регуляризацию

**С высоким variance (overfitting):**
- Больше данных (самое мощное средство)
- Регуляризация (L1, L2, dropout)
- Упростить модель (уменьшить количество параметров)
- Кросс-валидация для настройки гиперпараметров
- Ансамбли (RandomForest вместо одного дерева — снижает variance)

## Нормальное распределение

Самый важный закон распределения в статистике. Многие методы ML предполагают, что данные распределены нормально (хотя на практике это часто нарушается).

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

# Сгенерируем нормальное распределение
data = np.random.normal(loc=100, scale=15, size=1000)
# loc — среднее (центр колокола)
# scale — стандартное отклонение (ширина колокола)

# Эмпирическое правило (правило 68-95-99.7):
mean = np.mean(data)      # ≈ 100
std = np.std(data, ddof=1)  # ≈ 15

# В пределах 1σ (1 std): ~68% данных
# В пределах 2σ: ~95% данных
# В пределах 3σ: ~99.7% данных
within_1 = np.mean(np.abs(data - mean) < std)      # ~0.68
within_2 = np.mean(np.abs(data - mean) < 2 * std)  # ~0.95
```

**Почему нормальное распределение так важно?** Многие явления в природе примерно нормальны (рост, вес, IQ). Но в ML данные редко бывают нормальными (зарплаты, клики, время на сайте — обычно смещены). Однако **средние значений** (если брать много выборок) будут нормальными — это центральная предельная теорема.

### Центральная предельная теорема (ЦПТ)

Главная теорема статистики. Звучит сложно, но суть проста:

**Если взять много выборок из любого распределения и посчитать среднее каждой выборки — эти средние будут распределены нормально.**

Неважно, как выглядит исходное распределение — хоть равномерное, хоть экспоненциальное. Средние выборок из него подчиняются нормальному закону. Чем больше размер выборки, тем ближе к нормальному.

```python
# Демонстрация ЦПТ
# Берём экспоненциальное распределение (совсем не нормальное)
population = np.random.exponential(scale=5, size=100000)

# Берём 1000 выборок, считаем среднее каждой
sample_means = []
for _ in range(1000):
    sample = np.random.choice(population, size=50)
    sample_means.append(np.mean(sample))

# Распределение sample_means — нормальное (колокол)!
# Среднее этих средних ≈ среднему популяции
```

**Зачем это надо в ML?** ЦПТ объясняет, почему MSE (Mean Squared Error) работает, почему доверительные интервалы строятся через нормальное распределение, почему t-тесты валидны. Мы можем делать выводы о генеральной совокупности, имея только одну выборку.

## Доверительные интервалы

Мы посчитали среднее по выборке — например, средняя цена дома 5 млн. Но мы понимаем: на другой выборке было бы другое среднее. Где же "настоящее" среднее всей совокупности?

**Доверительный интервал** — это диапазон, в котором с заданной вероятностью (обычно 95%) находится истинное значение параметра совокупности.

```python
import scipy.stats as stats
import numpy as np

data = np.random.normal(100, 15, 100)  # выборка 100 элементов
mean = np.mean(data)
se = stats.sem(data)  # стандартная ошибка среднего (std / sqrt(n))

# 95% доверительный интервал
confidence = 0.95
t_critical = stats.t.ppf((1 + confidence) / 2, df=len(data) - 1)
ci_low = mean - t_critical * se
ci_high = mean + t_critical * se
print(f"95% ДИ: [{ci_low:.2f}, {ci_high:.2f}]")
```

**Как интерпретировать:** "Если бы мы провели эксперимент 100 раз, в 95 случаях истинное среднее было бы внутри такого интервала". Неправильно говорить "с вероятностью 95% истинное среднее лежит в этом интервале" — это распространённая ошибка. Интервал либо содержит истину, либо нет. 95% — это про **долгосрочную частоту**, а не про конкретный интервал.

### От чего зависит ширина интервала

1. **Размер выборки (n):** чем больше n, тем уже интервал (делим на sqrt(n))
2. **Разброс данных (std):** чем больше разброс, тем шире интервал
3. **Уровень доверия:** 99% интервал шире, чем 95% (мы хотим быть увереннее → платим шириной)

```python
# Связь: ДИ ~ t * std / sqrt(n)
# Чтобы интервал стал в 2 раза уже, нужно увеличить выборку в 4 раза!
```

### Как использовать в ML

Доверительные интервалы нужны, чтобы понимать, насколько мы можем доверять метрикам модели:

```python
from sklearn.metrics import accuracy_score
from sklearn.model_selection import cross_val_score

# Вместо одного числа accuracy получаем разброс
scores = cross_val_score(model, X, y, cv=10)  # 10 accuracy
mean_acc = np.mean(scores)
se_acc = stats.sem(scores)

ci_low = mean_acc - stats.t.ppf(0.975, 9) * se_acc
print(f"Accuracy: {mean_acc:.3f} ± {ci_low:.3f} (95% ДИ)")
```

Если доверительные интервалы двух моделей пересекаются — разница может быть случайной, а не значимой.

## P-value и проверка гипотез

### Нулевая и альтернативная гипотезы

Любая проверка гипотез начинается с двух утверждений:

- **H₀ (нулевая гипотеза):** "ничего не произошло", "разницы нет", "изменение не работает"
- **H₁ (альтернативная гипотеза):** "разница есть", "изменение работает"

Пример: новое лекарство.
- H₀: лекарство не влияет на давление (эффект = 0)
- H₁: лекарство снижает давление (эффект ≠ 0)

Мы никогда не "доказываем" H₁ напрямую. Мы смотрим: "насколько данные противоречат H₀?" Если противоречие сильное — отвергаем H₀ в пользу H₁.

### Что такое p-value

**P-value** — это вероятность получить такие же (или ещё более экстремальные) данные, **при условии, что H₀ верна**.

Простыми словами: "если на самом деле ничего не изменилось (H₀), то насколько необычно то, что мы наблюдаем?"

```python
# Пример: монетка. Подбросили 100 раз, выпало 60 орлов.
# H₀: монетка честная (p = 0.5)
# Вопрос: 60 орлов — это нормально для честной монетки?

from scipy import stats

# Биномиальный тест
p_value = stats.binom_test(60, n=100, p=0.5)
print(p_value)  # ~0.057 — вероятность получить 60+ орлов из 100 бросков
```

**Интерпретация p-value:**

| p-value | Что значит |
|---------|------------|
| > 0.05 | Данные не противоречат H₀. Недостаточно доказательств. |
| < 0.05 | Данные достаточно необычны при H₀. Отвергаем H₀. |
| < 0.01 | Сильное доказательство против H₀. |
| < 0.001 | Очень сильное доказательство. |

### Ошибки первого и второго рода

| | H₀ верна | H₁ верна |
|--|----------|----------|
| **Не отвергаем H₀** | ✅ Правильно | ❌ Ошибка II рода (β) — пропустили эффект |
| **Отвергаем H₀** | ❌ Ошибка I рода (α) — ложная тревога | ✅ Правильно |

- **Ошибка I рода (α):** мы сказали, что эффект есть, а его нет. Уровень значимости α = 0.05 — это наша "толерантность" к ложной тревоге. В 5% случаев мы будем видеть эффект там, где его нет.
- **Ошибка II рода (β):** мы сказали, что эффекта нет, а он есть. **Мощность теста** = 1 - β — вероятность обнаружить эффект, если он действительно есть.

**Почему это важно в ML?** При сравнении моделей/гиперпараметров мы делаем множество тестов. Если сравнивать 20 моделей с порогом p < 0.05, в среднем одна из них покажет "значимое" улучшение случайно (из-за множественной проверки). Это проблема множественного тестирования.

### Доверительный интервал и p-value — одно и то же

Они дают одинаковую информацию, просто с разных сторон:

```python
# Допустим, мы дали таблетку 100 людям
# Давление упало в среднем на 10 единиц
# Стандартная ошибка = 4

effect = 10
se = 4

# 95% доверительный интервал
ci = (effect - 1.96 * se, effect + 1.96 * se)  # (2.16, 17.84)

# p-value для H₀: эффект = 0
t_stat = effect / se   # 2.5
p_value = 2 * (1 - stats.norm.cdf(t_stat))  # ~0.012
```

**Версия p-value:** p = 0.01 < 0.05 → эффект значим.
**Версия ДИ:** 95% ДИ = [2.16; 17.84]. Внутри интервала нет нуля → эффект значим.

Вывод один и тот же. Доверительный интервал даёт дополнительную информацию — **размер эффекта** и его разброс. P-value даёт только "есть или нет".

## T-тест (Стьюдента)

Самый популярный тест для сравнения двух групп. Используется, когда данных мало и дисперсия неизвестна (почти всегда).

### Одно-выборочный t-тест

Сравниваем среднее выборки с известным значением:

```python
# Есть выборка роста студентов, хотим проверить, отличается ли
# средний рост от 170 см

heights = [168, 172, 165, 178, 171, 169, 175, 170, 167, 173]
t_stat, p_value = stats.ttest_1samp(heights, popmean=170)
print(f"t = {t_stat:.3f}, p = {p_value:.3f}")
# Если p < 0.05 — средний рост значимо отличается от 170
```

### Двух-выборочный t-тест (сравнение двух групп)

Сравниваем средние двух групп — например, контроль и тест в A/B эксперименте:

```python
# Время на сайте: группа A (контроль) vs группа B (новая версия)
group_a = [120, 135, 110, 145, 130, 125, 140, 115]
group_b = [145, 155, 140, 160, 150, 148, 165, 142]

t_stat, p_value = stats.ttest_ind(group_a, group_b)
print(f"t = {t_stat:.3f}, p = {p_value:.3f}")

# Интерпретация:
# Среднее A: 127.5, Среднее B: 150.6
# Разница: 23.1 секунды
# p < 0.05 → разница статистически значима
```

**Важные допущения t-теста:**
1. Данные примерно нормальны
2. Дисперсии групп примерно равны (если нет — используйте `ttest_ind(equal_var=False)`)
3. Наблюдения независимы (нельзя тестировать одних и тех же людей до/после как независимые группы)

```python
# Если дисперсии явно разные — Welch's t-test
t_stat, p_value = stats.ttest_ind(group_a, group_b, equal_var=False)
```

## Корреляция

Корреляция показывает, насколько две переменные связаны линейно.

### Коэффициент корреляции Пирсона (r)

- r = 1: идеальная положительная связь
- r = -1: идеальная отрицательная связь
- r = 0: нет линейной связи

```python
x = [1, 2, 3, 4, 5]
y = [2, 4, 6, 8, 10]   # идеальная линейная связь
r, p_value = stats.pearsonr(x, y)
print(f"r = {r:.3f}, p = {p_value:.3f}")  # r = 1.0, p = 0.0
```

**Важно:** корреляция ≠ причинность. Пример: продажи мороженого и утопления коррелируют, но мороженое не вызывает утопления — есть третья переменная (жара), которая влияет на оба показателя.

```python
# Пример ложной корреляции
ice_cream = [100, 120, 140, 160, 180, 200]
drownings = [10, 12, 15, 18, 20, 22]
r, p = stats.pearsonr(ice_cream, drownings)
# r ≈ 0.99, p ≈ 0.0001 — высокая значимая корреляция
# Но причинной связи нет!
```

### Корреляция Спирмена (ранговая)

Используется, когда связь нелинейная (но монотонная) или данные не нормальны:

```python
x = [1, 2, 3, 4, 5, 6, 7, 8]
y = [1, 4, 9, 16, 25, 36, 49, 64]  # квадратичная зависимость

r_pearson, _ = stats.pearsonr(x, y)     # ≈ 0.96 (не идеально, хотя связь чёткая)
r_spearman, _ = stats.spearmanr(x, y)   # = 1.0 (монотонная — идеально!)
```

## Когда что использовать: шпаргалка

| Задача | Метод |
|--------|-------|
| Сравнить средние двух групп | t-тест |
| Сравнить два распределения (не нормальные) | Mann-Whitney U |
| Сравнить доли (конверсия) | z-тест для пропорций |
| Сравнить >2 групп | ANOVA |
| Связь между двумя числовыми признаками | Корреляция Пирсона |
| Связь между порядковыми признаками | Корреляция Спирмена |
| Оценить параметр с разбросом | Доверительный интервал |
| Проверить гипотезу | p-value |
| Есть ли эффект вообще | t-тест / p-value |
| Какой размер эффекта | Доверительный интервал |
| Насколько точна модель | Bias-Variance анализ |
| Переобучена ли модель | Высокий variance (train >> test) |
| Недообучена ли модель | Высокий bias (train ≈ test, оба плохие) |

## Связь статистики и ML: главное

1. **Bias-Variance** — это диагноз модели. Высокий bias → усложняем. Высокий variance → упрощаем или добавляем данные.

2. **Доверительный интервал** — говорим не "accuracy = 85%", а "accuracy = 85% ± 3% (95% ДИ)". Это честнее и полезнее при сравнении.

3. **P-value** — отвечает на вопрос "разница есть или случайность?". Но не говорит, насколько велика разница. Для размера эффекта нужен ДИ.

4. **ЦПТ** — почему мы вообще можем что-то утверждать по выборке. Среднее выборки → нормальное → можно строить интервалы.

5. **Корреляция ≠ причинность** — самая частая ошибка. Две метрики могут коррелировать, но не влиять друг на друга.

6. **Ошибка I рода** — найдём "значимый" признак случайно, если перебирать много признаков. Всегда учитывайте множественную проверку.
