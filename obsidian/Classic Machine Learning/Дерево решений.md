> [!info] Зачем вообще деревья решений
> Дерево решений — это алгоритм, который пошагово задаёт вопросы вида "признак X больше/меньше Y?" или "признак X равен такому-то значению?", деля данные на всё более однородные группы. В итоге получается структура: корень (все данные) → внутренние узлы (условия-разбиения) → листья (финальное предсказание класса или значения). Дерево привлекательно тем, что его легко интерпретировать — можно буквально прочитать путь от корня до листа и понять, почему модель сделала такой вывод. Это отличает деревья от "чёрных ящиков" типа нейросетей.

#### Gini Impurity

Метрика изменения дерева. Она измеряет насколько чистым является информация в наборе классов. С точки зрения задач классификации, мы можем воспринимать это как однородность классов.

> [!note] Что значит формула
> Общая идея формулы Gini: для каждого класса считаем долю объектов этого класса в узле (p), умножаем на (1 - p) — то есть на вероятность "не угадать" этот класс, — и суммируем по всем классам. Если в узле один класс — Gini = 0 (узел идеально чистый, дальше делить не нужно). Если классы поровну — Gini максимален (наибольшая "путаница"). Поэтому при построении дерева мы всегда хотим **минимизировать** Gini Impurity — это и есть критерий, по которому выбирается, как лучше разбить данные.

Допустим начнём строить дерево и разбиваем дерево данных с 4 точками и 2 классами. Вычислим чистоту для наших данных. 
(2/4)(1 - 2/4) = 0.25 - Красный класс
(2/4)(1 - 2/4) = 0.25 - Синий класс
0.25 + 0.25 = 0.5 - Gini Impurity (максимально возможное значение, идеальный баланс классов)
Если будет полный дизбаланс классов, где какой то класс и вовсе отсутствует, то наша метрика выдаст - 0

Именно на основе данной метрике мы будем разделять наши данные в дереве и пытаться минимизировать данную метрику в листовых узлах (конечных)
Сделаем это на следующем рисунке
![[Pasted image 20241031195821.png]]

Высчитаем нашу метрику на признаке X - url link

Левый узел - (2/3)(1 - 2/3) + (1/3)(1 - 1/3) = 0.44
Правый узел - (1/4)(1 - 1/4) + (3/4)(1 - 3/4) = 0.375
Для всего признака - Всего писем = 7. Слева 3 письма, Справа 4 письма
(3/7)* 0.44  + (4/7)* 0.375 = 0.403

> [!tip] Почему взвешиваем по числу объектов
> Обрати внимание: финальное число 0.403 — это не просто среднее Gini левого и правого узла, а **взвешенное** среднее (умножаем на долю объектов, попавших в каждый узел). Это важно: разбиение, где в одном узле 1 объект с Gini=0, а в другом 1000 объектов с Gini=0.5, не должно считаться отличным только из-за одного чистого, но крошечного узла. Взвешивание учитывает размер каждой группы.
>
> Полученное значение 0.403 называется **Weighted Gini** этого конкретного разбиения по признаку X. Дальше это число сравнивается с Weighted Gini других возможных разбиений (по другим признакам или другим порогам того же признака) — и выбирается то разбиение, у которого это значение наименьшее.

> [!note] Непрерывные признаки подробнее
> Имеется в виду, что для числового признака (например, "возраст" или "сумма платежа") нельзя просто перебрать все возможные значения — их бесконечно много. Поэтому: 
> 1. Сортируем все значения признака по возрастанию.
> 2. Берём пороги ровно между соседними значениями (например, между 1-й и 2-й строкой, между 2-й и 3-й и так далее) — таких порогов будет на один меньше, чем строк данных.
> 3. Для каждого порога считаем Weighted Gini, как будто это бинарное разбиение "меньше порога / больше порога".
> 4. Выбираем порог с минимальным Gini.
>
> Это самая ресурсоёмкая часть построения дерева — для каждого числового признака нужно проверить много порогов, а для каждого порога — пересчитать метрику.

> [!note] Категориальные признаки подробнее
> Для категориального признака (например, "цвет: красный/синий/зелёный") дерево не просто делит "значение А против всех остальных" по очереди — оно также проверяет **группы** значений (например, "красный и зелёный" против "синий"). Для признака с k категориями число возможных способов разбить их на две группы растёт экспоненциально (2^(k-1) - 1 вариантов), поэтому на практике существуют разные эвристики, чтобы не перебирать всё буквально — например, сортировка категорий по доле целевого класса перед перебором.

Дерево крайне любит переобучаться, но этого можно избежать различными методами. У деревьев есть очень много различных гиперпараметров, глубина дерева(отвечает за максимальную глубину по сплитам признака в дереве), количество окончательных листьев (это последние узлы в дереве, отвечает за ключевой признак), так же гиперпараметр отвечающих за значение нашей метрики, если допустим разбиение слишком сильно сократит нашу метрику, то мы говорим, что не стоит разбивать, дабы не переобучиться.

> [!warning] Почему дерево переобучается
> Без ограничений дерево будет расти, пока в каждом листе не окажется один-единственный объект (Gini = 0 везде). Это значит, что дерево идеально "запомнило" обучающую выборку, включая весь шум и случайные выбросы — а не выучило общую закономерность. На новых данных такое дерево покажет плохой результат. Это классический пример высокой **variance** (дисперсии) модели в дилемме bias-variance trade-off.

> [!example] Основные гиперпараметры против переобучения
> - **max_depth** — максимальная глубина дерева (то, что уже описано в тексте).
> - **min_samples_split** — минимальное число объектов в узле, чтобы его вообще можно было делить дальше.
> - **min_samples_leaf** — минимальное число объектов, которое должно остаться в каждом листе после разбиения.
> - **max_leaf_nodes** — это и есть "количество окончательных листьев" из текста — ограничивает общее число листьев в дереве.
> - **min_impurity_decrease** — это и есть гиперпараметр "отвечающий за значение метрики" — минимальное необходимое снижение Gini, чтобы разбиение считалось оправданным.
> - **Pruning (обрезка дерева)** — альтернативный подход: сначала вырастить большое дерево, а потом отрезать от него ветви, которые не дают значимого улучшения на отдельной валидационной выборке (например, cost-complexity pruning).

> [!summary] Альтернатива Gini — Entropy
> Кроме Gini Impurity, есть вторая популярная метрика — **Entropy** (энтропия), пришедшая из теории информации:
> Entropy = -Σ p·log₂(p) по всем классам
> Она работает похожим образом (0 — идеальная чистота, максимум — при равных долях классов), но штрафует смешанные узлы немного сильнее, чем Gini, и требует вычисления логарифма, поэтому считается чуть медленнее. На практике обе метрики почти всегда приводят к похожим деревьям — Gini выбирают чаще просто из-за скорости вычислений (sklearn использует Gini по умолчанию).

> [!info] Деревья для регрессии
> Всё описанное выше — это **дерево классификации** (predict класса). Для **регрессии** (предсказание числа, например цены) логика та же — разбиваем данные по тем же принципам, — но вместо Gini Impurity используется **дисперсия (variance)** или **MSE** внутри узла: ищем разбиение, которое сильнее всего снижает разброс значений целевой переменной в дочерних узлах. Предсказанием в листе становится среднее значение целевой переменной среди объектов, попавших в этот лист.

> [!quote] Куда это ведёт дальше
> Одиночное дерево решений редко используется само по себе в продакшене именно из-за склонности к переобучению. Знание этой темы — фундамент для понятия **ансамблей деревьев**:
> - **Random Forest** — строит много деревьев на случайных подвыборках данных и случайных подмножествах признаков, усредняя их предсказания (снижает variance).
> - **Gradient Boosting** (XGBoost, LightGBM, CatBoost) — строит деревья последовательно, каждое новое дерево исправляет ошибки предыдущих.
> Понимание Gini Impurity и механики разбиения, описанной в этой заметке, напрямую переносится на оба этих метода.
