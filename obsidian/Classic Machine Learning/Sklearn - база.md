___

Scikit-learn (sklearn) - это библиотека машинного обучения, построенная вокруг единой философии: все ключевые этапы работы с данными оформлены как объекты с одинаковой структурой методов. Такой подход позволяет собирать сложные эксперименты из кирпичиков, не запоминая разрозненные функции. В основе лежат три типа объектов: **Estimator** (любая модель или преобразователь, который учится на данных), **Transformer** (преобразователь, имеющий методы `fit` и `transform`) и **Predictor** (модель, способная делать предсказания через `predict` или `predict_proba`).

**Обучение с учителем** - Обучение модели когда у нас есть Целевой признак и мы подаём его так же в модель вместе со всеми другими данными, на основе которых он и пытается предсказать Целевой признак
**Обучение без учителя** - У нас нету целевого признака, подаются все признаки, дальше по обучению вы познакомитесь с парочкой таких методов (KNN и Кластеризация)

Главное правило: любой объект, который обучается, использует метод `fit(X, y)` (для обучения с учителем) или `fit(X)` (для обучения без учителя). Преобразователи дополнительно дают `transform(X)`, а модели - `predict(X)`. Благодаря этому мы можем объединять последовательности шагов в конвейеры (`Pipeline`) и применять кросс-валидацию или подбор гиперпараметров ко всему процессу сразу, избегая утечек данных.

Посмотрим на самый простой базовый Пайплайн на данной библиотеке, будет как пример работы структуры написания пайплайнов на Sklearn. Живой пример, демонстрирующий интерфейс на задаче классификации ирисов. Обратите внимание на единообразие вызовов: сначала создаём объект, затем обучаем, затем предсказываем.

```python
 # === Импорты ===
from sklearn.datasets import load_iris                # стандартный датасет
from sklearn.model_selection import train_test_split  # разделение данных
from sklearn.linear_model import LogisticRegression   # модель (Estimator + Predictor)
from sklearn.metrics import accuracy_score            # метрика


# 1. Загрузка данных
# load_iris возвращает объект Bunch, похожий на словарь
iris = load_iris() # ТУТ У НАС ДАТАФРЕЙМ ПАНДАС С КОТОРЫМ ВЫ УЖЕ ЗНАКОМЫ
X = iris.data          # матрица признаков (150, 4)
y = iris.target        # вектор меток (150,)

# 2. Разделение на обучающую и тестовую выборки
# stratify=y сохраняет пропорции классов в обеих частях
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y)
    
# 3. Создание модели (Estimator)
# LogisticRegression — это и Estimator, и Predictor
model = LogisticRegression(max_iter=200, random_state=42) # с данной моделью вы более подробно познакомитесь дальше в блоке Логистическая регрессия, пока просто важно понимать, что это модель которая обучается на данных и предсказывает целевую (y)

# 4. Обучение модели
# fit() принимает признаки и целевую переменную, возвращает сам объект
model.fit(X_train, y_train)

# 5. Предсказание на тесте
# predict() выдаёт метки классов, predict_proba() — вероятности
y_pred = model.predict(X_test)

# 6. Оценка качества
# accuracy_score — функция, не метод модели
acc = accuracy_score(y_test, y_pred)
print(f"Accuracy: {acc:.3f}")

# Дополнительно: модель имеет доступ к изученным параметрам
print("Коэффициенты модели (одна строка на класс):")
print(model.coef_)          # (3, 4) — по числу классов и признаков
print("Intercept (свободный член):", model.intercept_)

```

**Что здесь важно увидеть:**
- `fit()` обучает и запоминает состояние объекта.
- После `fit()` модель готова к предсказаниям (`predict`, `predict_proba`, `score`).
- Трансформеры работают аналогично: вызов `fit_transform(X_train)` на обучающей выборке и просто `transform(X_test)` на тестовой.
- Все оценки (метрики) принимают истинные и предсказанные значения, а не встроены в модель.

Такой шаблон будет повторяться с любым алгоритмом, будь то случайный лес, градиентный бустинг или PCA. Единственное различие - гиперпараметры, которые передаются при создании объекта. Дальше в заметке мы детально разберём, как масштабировать признаки, работать с категориальными переменными и собирать всё это в конвейер, чтобы никогда не переобучаться на тесте.


### Подготовка данных
___

Перед тем как применять модели, данные необходимо загрузить, осмотреть и правильно разделить на обучающую и тестовую выборки. Scikit-learn предоставляет набор встроенных датасетов, а также инструменты для создания синтетических данных - это удобно для экспериментов. Ключевое правило: тестовая выборка должна быть изолирована с самого начала, и её нельзя использовать ни для обучения, ни для принятия решений о предобработке. Разделение выполняется функцией `train_test_split`, которая и станет нашим главным инструментом на этом этапе.

Ниже разберём загрузку, беглый осмотр и разделение на примере задачи классификации ирисов.

#### 2.1 Загрузка встроенного датасета
Scikit-learn содержит модуль `sklearn.datasets` с популярными наборами. Функции `load_*` возвращают маленькие датасеты в виде объектов `Bunch`, похожих на словари.
```python
from sklearn.datasets import load_iris

iris = load_iris()          # объект DataFrame
X = iris.data               # numpy-массив признаков (150, 4)
y = iris.target             # numpy-массив меток (150,)
feature_names = iris.feature_names  # названия признаков
target_names = iris.target_names    # названия классов
```

Если вы хотите взять свои данные, а не импортировать их, то это всегда можно сделать при помощи пандаса и работать с объектом датафрейм весь пайплайн:
```python
import pandas as pd

data = pd.read_csv("data.csv")      # объект DataFrame
X = data.data               # numpy-массив признаков (150, 4)
y = data.target             # numpy-массив меток (150,)
feature_names = data.feature_names  # названия признаков

```

#### 2.2 Быстрый осмотр данных
До разделения полезно бросить взгляд на форму массива и распределение классов — это помогает вовремя заметить ошибки (например, перепутанные строки и столбцы).

```python
# Форма матрицы признаков и вектора меток
print(X.shape)   # (150, 4) — 150 наблюдений, 4 признака
print(y.shape)   # (150,)  — одномерный массив меток
```

Для классификации критично проверить баланс классов:
```python
import pandas as pd

# value_counts() удобнее использовать через pandas
print(pd.Series(y).value_counts())
# Ожидаем равное количество по 50 каждого класса для iris
```
Если классы несбалансированны, это повлияет на стратегию разделения (стратификация) и выбор метрик.

#### 2.3 Разделение на train и test
Основной инструмент - `train_test_split`. Он случайно перемешивает данные и делит их в заданной пропорции. Мы всегда фиксируем `random_state` для воспроизводимости, а для классификации обязательно используем `stratify=y`, чтобы сохранить пропорции классов в обеих выборках.
```python
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.3,       # 30% данных уйдёт в тест
    random_state=42,     # фиксируем генератор случайных чисел
    stratify=y           # сохраняем распределение классов
)
```

Если вы параллельно с записями пишите код и проверяете его вывод, то для демонстрации будет полезно. Проверим, что пропорции действительно сохранились:
```python
print("Train class distribution:")
print(pd.Series(y_train).value_counts(normalize=True))
print("\nTest class distribution:")
print(pd.Series(y_test).value_counts(normalize=True))
```
Результат покажет примерно одинаковые доли классов (0.333 для каждого), в отличие от случайного разделения без `stratify`, где мог возникнуть перекос.

#### Что важно запомнить

- `train_test_split` всегда вызывается **до** любого масштабирования, заполнения пропусков или кодирования категорий. Иначе информация из теста «утечёт» в обучение.
- Для регрессии параметр `stratify` не используется, и разбиение происходит чисто случайно.
- В реальных проектах часто выделяют ещё и валидационную выборку. Это можно сделать повторным вызовом `train_test_split` на `X_train`, либо использовать `ShuffleSplit` или кросс-валидацию (позже).
- Фиксируйте `random_state` числом во всех экспериментах, иначе сравнение разных моделей потеряет смысл.

Теперь, имея изолированные `X_train` и `X_test`, можно безопасно переходить к предобработке признаков.

### Предобработка данных (трансформеры, нет, не те что в LLM)
___
Модели машинного обучения понимают только числа, поэтому категориальные переменные нужно кодировать, а числовые часто масштабировать или очищать от пропусков. Scikit-learn предоставляет для этого объекты-трансформеры с одинаковым интерфейсом: метод `fit()` изучает параметры преобразования (среднее, максимальное значение, набор категорий и т.д.), а `transform()` применяет изученное к данным. Критически важно: `fit()` (или `fit_transform()`) вызывается только на обучающей выборке, а тестовая только трансформируется через `transform()` это предотвращает утечку информации.

Ниже разберём ключевые трансформеры по типам признаков, а затем научимся объединять их в единый блок с помощью `ColumnTransformer`.

#### 3.1 Масштабирование числовых признаков
Масштабирование приводит признаки к одному диапазону, что необходимо для методов, чувствительных к величине значений (SVM, нейронные сети, линейные модели с регуляризацией, PCA). Три основных варианта:

- `StandardScaler` — вычитает среднее и делит на стандартное отклонение (среднее = 0, std = 1).
- `MinMaxScaler` — преобразует в заданный диапазон (по умолчанию [0, 1]).
- `RobustScaler` — использует медиану и квартили, устойчив к выбросам.

Пример с `StandardScaler` на наших числовых данных `X_train`:
```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()             # создаём объект-трансформер
X_train_scaled = scaler.fit_transform(X_train)  # на трейне: fit + transform
X_test_scaled = scaler.transform(X_test)        # на тесте: только transform, если сделать тут fit, но то данные уже будут отличаться от X_train статистически! Представим это так, что Тестовые данные, это то, чего модель никогда не видела, она обучилась и через неделю пришли тестовые данные, надо предсказать, но модель училась только на статистике тренировочных, в ней не должно быть информации о статистике тестовой выборки.

# Проверка: средние должны быть близки к 0, стандартные отклонения — к 1
print(X_train_scaled.mean(axis=0).round(2))   # ~ [0. 0. 0. 0.]
print(X_train_scaled.std(axis=0).round(2))    # ~ [1. 1. 1. 1.]
```

Объект `scaler` сохранил среднее и стандартное отклонение для каждого признака:
```python
print(scaler.mean_)        # средние исходных признаков
print(scaler.scale_)       # std, на которые поделили
```

#### 3.2 Обработка пропущенных значений
Пропуски в числовых колонках можно заполнять, не теряя строки. Базовый инструмент — `SimpleImputer`. Ну либо как мы знаем, всё это можно заполнить при помощи самого Pandas в датафрейме.
```python
import numpy as np
from sklearn.impute import SimpleImputer

# Допустим, в X_train есть пропуски (NaN)
X_train_with_nan = X_train.copy()
X_train_with_nan[0, 0] = np.nan   # искусственно внесём один пропуск

imputer = SimpleImputer(strategy='mean')  # стратегия: среднее по столбцу
X_train_imputed = imputer.fit_transform(X_train_with_nan)
```

Параметр `strategy` может быть `'median'`, `'most_frequent'` или `'constant'` (тогда задаётся `fill_value`). Для более сложного заполнения (например, KNN) есть `KNNImputer`. Когда познакомитесь с алгоритмом KNN, то прекрасно поймёте что делает это сложное заполнение. (Я его как то пробовал на более миллионе данных, часов 6 заполнял всё, ужас)

Важно: `fit()` вычисляет средние только по обучающей выборке; тестовые пропуски заполняются тренировочными средними — это тоже исключает утечку.
```python
# К тесту применяем уже обученный imputer
X_test_imputed = imputer.transform(X_test)
```

#### 3.3 Полиномиальные признаки (создание новых)
Иногда полезно добавить взаимодействия и степени признаков, например, для линейной регрессии, чтобы уловить нелинейности.

```python
from sklearn.preprocessing import PolynomialFeatures

poly = PolynomialFeatures(degree=2, include_bias=False)  # добавляем квадраты и попарные произведения
X_train_poly = poly.fit_transform(X_train_scaled)
print(f"Было признаков: {X_train.shape[1]}, стало: {X_train_poly.shape[1]}")
```
Этот трансформер тоже применяется после разделения, чтобы избежать генерации признаков на основе тестовых данных.

#### 3.4 Категориальные признаки
Для превращения текстовых категорий в числа есть два основных способа: One-Hot Encoding и Ordinal Encoding. Начнём с One-Hot, так как он не привносит ложного порядка.

Для превращения текстовых категорий (большинство моделей с ними не умеют работать) в числа есть два основных способа: One-Hot Encoding и Ordinal Encoding. Начнём с One-Hot, так как он не привносит ложного порядка.

import pandas as pd
from sklearn.preprocessing import OneHotEncoder
```python
# Представим DataFrame с одним категориальным столбцом
train_df = pd.DataFrame({'city': ['Moscow', 'London', 'Paris', 'London']})
test_df = pd.DataFrame({'city': ['Paris', 'Berlin']})  # в тесте встречается новая категория

encoder = OneHotEncoder(
    sparse_output=False,      # чтобы получить обычную матрицу, а не разреженную
    handle_unknown='ignore'   # НЕ падать при встрече новой категории в тесте
)
X_train_ohe = encoder.fit_transform(train_df[['city']])
print(X_train_ohe)
# Londn | Paris | Moscow
# [[0.     0.       1.]  ← Moscow
#  [1.     0.       0.]  ← London
#  [0.     1.       0.]  ← Paris
#  [1.     0.       0.]] ← London
```

Обученный энкодер запомнил категории `['London', 'Paris', 'Moscow']` (порядок можно увидеть в `encoder.categories_`). Теперь трансформируем тест:
```python
X_test_ohe = encoder.transform(test_df[['city']])
print(X_test_ohe)
# [[0. 1. 0.]  ← Paris (категория знакома)
#  [0. 0. 0.]] ← Berlin — полностью нулевая строка, потому что handle_unknown='ignore'
```

Без `handle_unknown='ignore'` мы получили бы ошибку `ValueError` на этапе трансформации теста. Поэтому **всегда** указывайте этот параметр при обучении на категориях с потенциально новым значением в будущем.

`OrdinalEncoder` используют, когда у категорий есть естественный порядок (например, `['low', 'medium', 'high']`). Интерфейс аналогичный, но возвращает целые числа.

#### 3.5 Объединение трансформеров с ColumnTransformer
Реальные данные обычно содержат и числовые, и категориальные столбцы. Мы не должны обрабатывать их вручную по отдельности — есть `ColumnTransformer`, который применяет разные трансформеры к разным колонкам и склеивает результат 
(ОЧЕНЬ КРУТАЯ ВЕЩЬ КОТОРОЙ Я НИКОГДА САМ НЕ ПОЛЬЗОВАЛСЯ).
```python
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import MinMaxScaler

# Представим датасет со столбцами: age (число), income (число), city (категория), gender (категория)
train = pd.DataFrame({
    'age': [25, 40, 35],
    'income': [50000, 80000, 60000],
    'city': ['Moscow', 'London', 'Paris'],
    'gender': ['M', 'F', 'F']
})

# Определяем списки столбцов по типам
numeric_features = ['age', 'income']
categorical_features = ['city', 'gender']

preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numeric_features),        # числовые — масштабируем
        ('cat', OneHotEncoder(sparse_output=False, handle_unknown='ignore'), categorical_features)
    ]
)

X_train_processed = preprocessor.fit_transform(train)
print(X_train_processed)
# Вывод: первые 2 столбца — стандартизированные age и income,
#        затем one-hot векторы для city и gender
```

Чтобы не перечислять имена колонок вручную (особенно когда их десятки), удобно использовать `make_column_selector`:
```python
from sklearn.compose import make_column_selector

preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), make_column_selector(dtype_include=np.number)),
        ('cat', OneHotEncoder(sparse_output=False, handle_unknown='ignore'),
         make_column_selector(dtype_include=object))
    ]
)
```

Теперь трансформер автоматически выбирает все числовые колонки и все текстовые.

### Что важно запомнить

- **Разделение `fit` и `transform`:** на обучающей выборке используйте `fit_transform()` или отдельно `fit()` + `transform()`. На тестовой — только `transform()`.
- **Категории и неизвестные значения:** всегда ставьте `handle_unknown='ignore'` в `OneHotEncoder`, если в будущем могут появиться новые категории.
- **Sparse matrix:** по умолчанию `OneHotEncoder` выдаёт разреженную матрицу. Чтобы получить обычный массив, укажите `sparse_output=False`, либо используйте `model`, которая умеет работать с разреженными матрицами (например, `LogisticRegression`).
- **Порядок шагов:** сначала разделяем данные на train/test, потом настраиваем и применяем `ColumnTransformer` к train, и только потом к test.
- **Полиномиальные признаки и импутация:** также встраиваются в `ColumnTransformer` к нужным столбцам.

Теперь у нас есть все компоненты для сборки полноценного пайплайна, который объединяет предобработку и модель в один объект.

### Пайплайны (Pipeline)
___

Предобработка данных и модель - это не изолированные этапы. Если масштабировать данные вручную отдельно от модели, легко совершить ошибку: применить разные трансформации к трейну и тесту, забыть обработать новые данные перед предсказанием или, что хуже всего, допустить утечку информации из теста в обучение (например, вычислив среднее по всему датасету до разделения). Пайплайны решают эту проблему: они объединяют все шаги (трансформеры + модель) в единый объект, который можно обучить одной командой `fit()` и использовать для предсказания одной командой `predict()`. При этом всё, что находится внутри пайплайна, «учится» только на тренировочных данных, а при вызове `predict()` на новых данных применяет заученные трансформации и затем предсказывает.

#### 4.1 Простой Pipeline из двух шагов
Соберём конвейер из масштабирования (`StandardScaler`) и логистической регрессии. Имена шагов могут быть любыми, но должны быть строками без пробелов и использоваться для доступа к параметрам.

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ('scaler', StandardScaler()),            # шаг 1: масштабирование
    ('classifier', LogisticRegression(max_iter=200, random_state=42))  # шаг 2: модель
])
```

Теперь `pipe` - это один объект, который ведёт себя как модель. Вызовем `fit` на исходных (немасштабированных) данных:
```python
pipe.fit(X_train, y_train)
```

Под капотом происходит следующее:

- `X_train` передаётся в `StandardScaler`, который вычисляет среднее и std, затем масштабирует данные.
- Масштабированные данные поступают в `LogisticRegression`, которая обучается на них и на `y_train`.

Посмотрим на точность:
```python
accuracy = pipe.score(X_test, y_test)  # score автоматически трансформирует X_test и считает accuracy
print(f"Pipeline accuracy: {accuracy:.3f}")
```

Метод `predict()` тоже работает прозрачно:
```python
y_pred = pipe.predict(X_test)  # трансформация + предсказание
```

Таким образом, исключена ситуация, когда к тесту случайно применяется `fit_transform` вместо `transform` - пайплайн делает это сам правильно.

#### 4.2 Быстрое создание с make_pipeline
Если не хочется придумывать имена шагам, можно использовать `make_pipeline`, который назовёт их автоматически (в нижнем регистре по имени класса: `standardscaler`, `logisticregression`).
```python
from sklearn.pipeline import make_pipeline

pipe_auto = make_pipeline(StandardScaler(), LogisticRegression(max_iter=200, random_state=42))
pipe_auto.fit(X_train, y_train)
print(pipe_auto.score(X_test, y_test))
```
Это удобно для быстрых экспериментов, но в больших проектах осмысленные имена облегчают отладку и подбор гиперпараметров.

#### 4.3 Сборка полного пайплайна с ColumnTransformer
На практике данные состоят из столбцов разных типов. Объединим всё, что изучили: `ColumnTransformer` для предобработки и модель в единый `Pipeline`.

Представим датасет в виде DataFrame с числовыми и категориальными признаками:
```python
import pandas as pd
from sklearn.model_selection import train_test_split

# Создадим примерный датафрейм
df = pd.DataFrame({
    'age': [25, 40, 35, 50, 23],
    'income': [50000, 80000, 60000, 100000, 40000],
    'city': ['Moscow', 'London', 'Paris', 'London', 'Moscow'],
    'target': [1, 0, 1, 0, 1]
})

X = df[['age', 'income', 'city']]
y = df['target']

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.4, random_state=42, stratify=y)
```

Теперь строим пайплайн с препроцессором и классификатором:
```python
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression

# Числовые и категориальные колонки
numeric_cols = ['age', 'income']
categorical_cols = ['city']

# Препроцессор: к разным колонкам — разные трансформеры
preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numeric_cols),
        ('cat', OneHotEncoder(sparse_output=False, handle_unknown='ignore'), categorical_cols)
    ]
)

# Финальный пайплайн: препроцессор + модель
clf = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression(max_iter=200, random_state=42))
])

clf.fit(X_train, y_train)
print(f"Accuracy: {clf.score(X_test, y_test):.3f}")
```

Обратите внимание: `X_train` подаётся как DataFrame - `ColumnTransformer` сам вытащит нужные столбцы по именам. Если подать массив NumPy, то придётся указывать индексы колонок, что менее надёжно.

#### 4.4 Доступ к шагам пайплайна
Обученный пайплайн позволяет добраться до любого промежуточного шага для инспекции:
```python
# Получаем обученный StandardScaler
scaler = clf.named_steps['preprocessor'].named_transformers_['num']
print(scaler.mean_)

# Коэффициенты логистической регрессии
print(clf.named_steps['classifier'].coef_)
```

#### Что важно запомнить
- **Пайплайн предотвращает утечку данных** — весь `fit` (включая вычисление средних, категорий и прочего) происходит только на обучающей части, даже если мы обернули всё в один объект.
- **Имена шагов** нужны для тонкой настройки: через `set_params` можно менять гиперпараметры внутри пайплайна, обращаясь к ним как `имя_шага__параметр`. Это станет основой для `GridSearchCV`.
- **Всегда используйте пайплайны**, как только проект выходит за рамки игрушечного примера. Это делает код надёжным и переиспользуемым.
- **Сохраняйте обученный пайплайн целиком** (через `joblib`), а не отдельно скейлер и модель, чтобы не нарушить последовательность преобразований при загрузке.

Теперь у нас есть цельный объект, который можно передавать в кросс-валидацию и подбор гиперпараметров — этим и займёмся в следующем разделе.



### Модели (основные семейства)
___
Scikit-learn предоставляет десятки алгоритмов, но все они подчиняются единому интерфейсу: создали объект, вызвали `fit()`, получили предсказания через `predict()` (или `predict_proba()` для вероятностей). В этом разделе пройдёмся по ключевым семействам моделей, которые закрывают 90% практических задач. Для каждой - короткий контекст «когда применять», пример кода и важнейшие гиперпараметры.

#### 5.1 Линейные модели

Линейные модели - первый выбор, когда важны скорость и интерпретируемость. Они предполагают линейную зависимость между признаками и целевой переменной (для регрессии) или линейную разделяющую границу (для классификации).

**Регрессия:**
```python
from sklearn.linear_model import LinearRegression, Ridge, Lasso

# Обычная линейная регрессия
lr = LinearRegression()
lr.fit(X_train_scaled, y_train)
print("Коэффициенты:", lr.coef_)
```

`LinearRegression` использует метод наименьших квадратов без регуляризации. Если признаков много или они коррелируют, коэффициенты могут нестабильно оцениваться. Тогда на помощь приходят `Ridge` (L2-регуляризация) и `Lasso` (L1-регуляризация, зануляет часть коэффициентов):

```python
ridge = Ridge(alpha=1.0)   # alpha — сила регуляризации
ridge.fit(X_train_scaled, y_train)

lasso = Lasso(alpha=0.1)
lasso.fit(X_train_scaled, y_train)
print("Ненулевых коэффициентов у Lasso:", (lasso.coef_ != 0).sum())
```

**Классификация:** логистическая регрессия (несмотря на название, это классификатор).
```python
from sklearn.linear_model import LogisticRegression

logreg = LogisticRegression(
    penalty='l2',        # 'l1', 'l2', 'elasticnet', None
    C=1.0,               # обратная сила регуляризации (меньше — сильнее регуляризация)
    max_iter=200,
    multi_class='multinomial',  # для настоящей многоклассовой, иначе по умолчанию 'ovr'
    random_state=42
)
logreg.fit(X_train_scaled, y_train)
y_pred = logreg.predict(X_test_scaled)
y_proba = logreg.predict_proba(X_test_scaled)   # вероятности классов
```

### Что важно запомнить по всем моделям

- **Масштабирование:** для линейных моделей (с регуляризацией), SVM, K-Means, PCA — обязательно. Деревьям и ансамблям на основе деревьев оно не нужно.
- **Случайность:** фиксируйте `random_state` для воспроизводимости.
- **Параметры по умолчанию:** часто дают приемлемый результат, но для улучшения нужно подбирать (скоро узнаем как).
- **Интерпретация важности:** у линейных моделей — это коэффициенты (при масштабированных признаках), у деревьев и ансамблей — `feature_importances_`.
- **Все модели можно и нужно помещать в `Pipeline`:** после предобработки ставите модель, и весь конвейер — как одна модель. Это упрощает подбор гиперпараметров и внедрение в продакшен.

Теперь, зная основные алгоритмы, можно переходить к оценке качества и тонкому подбору гиперпараметров — об этом в следующем разделе.




### Оценка качества: метрики
___
Модель обучена и выдаёт предсказания - теперь нужно измерить, насколько они хороши. Scikit-learn предоставляет широкий набор метрик в модуле `sklearn.metrics`. Выбор метрики зависит от типа задачи (классификация или регрессия) и бизнес-контекста. Универсального «лучшего» показателя нет: accuracy может обманывать при несбалансированных классах, а RMSE чувствителен к выбросам. Поэтому важно понимать, что именно измеряет каждая метрика.

Все метрики следуют общему шаблону: функция принимает истинные значения (`y_true`) и предсказанные (`y_pred`, а для вероятностных - `y_score`) и возвращает число или отчёт. Ниже разберём основные метрики для классификации и регрессии.

#### 6.1 Метрики классификации
##### Accuracy (доля правильных ответов)

Самая простая метрика — `accuracy_score`. Она показывает долю совпадений, но бесполезна, если классы сильно неравномерны (например, 95% объектов класса 0 — модель, всегда предсказывающая 0, получит accuracy 0.95).

```python

from sklearn.metrics import accuracy_score
accuracy = accuracy_score(y_test, y_pred)
print(f"Accuracy: {accuracy:.3f}")

##### Матрица ошибок (confusion matrix)

Показывает расклад по каждому классу: сколько каких объектов куда отнесено.

python

from sklearn.metrics import confusion_matrix
cm = confusion_matrix(y_test, y_pred)
print("Confusion matrix:\n", cm)
# Для бинарной классификации строки — истинные классы, столбцы — предсказанные.
# [[TN, FP],
#  [FN, TP]]
```

На основе матрицы считаются производные метрики: точность, полнота, F1.

##### Точность, полнота и F1-мера
- **Precision** — сколько из предсказанных положительными действительно положительны.
- **Recall** — сколько положительных найдено.
- **F1-score** — гармоническое среднее precision и recall.

Можно получить все сразу с помощью `classification_report`:

```python

from sklearn.metrics import classification_report
print(classification_report(y_test, y_pred, target_names=target_names))

Вывод показывает precision, recall, f1-score и support для каждого класса, а также средние (`macro avg`, `weighted avg`).

Для получения отдельных чисел:

python

from sklearn.metrics import precision_score, recall_score, f1_score
precision = precision_score(y_test, y_pred, average='macro')  # или 'weighted', 'binary' и т.д.
recall = recall_score(y_test, y_pred, average='macro')
f1 = f1_score(y_test, y_pred, average='macro')
print(f"Precision: {precision:.3f}, Recall: {recall:.3f}, F1: {f1:.3f}")
```
Параметр `average` определяет способ усреднения по классам:

- `'binary'` — только для одного класса (по умолчанию для бинарной классификации).
- `'macro'` — среднее по классам без учёта размера класса.
- `'weighted'` — среднее, взвешенное по размеру класса.
- `'micro'` — считает общие TP, FP, FN по всем классам (равно accuracy для однозначной классификации).

##### ROC-AUC (площадь под ROC-кривой)
Метрика, оценивающая способность модели разделять классы на основе вероятностей. Требует вероятности (`predict_proba`), а не жёсткие метки.

```python

from sklearn.metrics import roc_auc_score, roc_curve
# Для бинарного случая берём вероятности положительного класса (второй столбец)
y_proba = model.predict_proba(X_test)[:, 1]
auc = roc_auc_score(y_test, y_proba)
print(f"ROC-AUC: {auc:.3f}")
# Кривая для графика (не обязательно в коде, но полезно знать)
fpr, tpr, thresholds = roc_curve(y_test, y_proba)
```
Для многоклассовой классификации нужно указать `multi_class` и `average`:

```python

# multi_class='ovr' (one-vs-rest) или 'ovo' (one-vs-one)
auc_multiclass = roc_auc_score(y_test, model.predict_proba(X_test), multi_class='ovr', average='macro')
```
#### 6.2 Метрики регрессии

##### Среднеквадратичная ошибка (MSE) и корень из неё (RMSE)
MSE — квадрат разности истинного и предсказанного, RMSE — корень из MSE, возвращает ошибку в единицах целевой переменной.

```python

from sklearn.metrics import mean_squared_error
mse = mean_squared_error(y_test, y_pred)
rmse = mean_squared_error(y_test, y_pred, squared=False)  # squared=False даёт RMSE
print(f"MSE: {mse:.3f}, RMSE: {rmse:.3f}")
```
##### Средняя абсолютная ошибка (MAE)
Среднее абсолютных разностей — менее чувствительна к выбросам, чем MSE.

```python

from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(y_test, y_pred)
print(f"MAE: {mae:.3f}")
```
##### Коэффициент детерминации R²
Показывает долю объяснённой дисперсии (1 — идеально, 0 — модель предсказывает среднее, может быть отрицательным).

```python

from sklearn.metrics import r2_score
r2 = r2_score(y_test, y_pred)
print(f"R²: {r2:.3f}")
```
#### 6.3 Встроенный метод score у моделей

Многие модели имеют метод `score()`, который вычисляет стандартную для этого типа задач метрику. Для классификаторов это accuracy, для регрессоров — R². Однако при несбалансированных классах accuracy малоинформативен, поэтому лучше явно вызывать подходящую метрику.

```python
# Встроенный метод (accuracy для классификации)
print(model.score(X_test, y_test))
# Явный вызов нужной метрики
print(f1_score(y_test, y_pred, average='macro'))
```
#### Что важно запомнить

- **Accuracy не подходит для несбалансированных данных.** Всегда смотрите на precision, recall, F1 или ROC-AUC.
- **ROC-AUC требует вероятностей,** а не меток. Используйте `predict_proba()` и берите вероятности нужного класса.
- **Для многоклассовой классификации** в метриках всегда уточняйте параметр `average` (macro, weighted, micro), иначе можно получить неожиданное значение.
- **В регрессии** смотрите на несколько метрик: MAE для устойчивости к выбросам, R² для интерпретации.
- **Не оценивайте модель на обучающей выборке**, кроме как для диагностики переобучения. Основная оценка — на тестовой выборке или через кросс-валидацию.
